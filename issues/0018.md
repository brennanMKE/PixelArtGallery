# 0018 — Let the user name an imported image instead of hardcoding "Imported Image"

| | |
|---|---|
| **Status** | open |
| **Module** | UI / ViewModels |
| **Platform** | All |
| **First seen** | 2026-06-25 |

## Description

Every imported image is labeled "Imported Image", so the gallery fills with identically-named items and they can't be told apart. The user needs a way to set (and later edit) the name.

## Expected behavior

When importing, the user can enter a name for the image — ideally prefilled with a sensible default (e.g. the source filename where the picker provides one, or "Imported Image" as a fallback). The chosen name is stored on the `GalleryItem` and shown in the list and detail views. Renaming an existing gallery item afterward would also be good.

## Actual behavior

The name is hardcoded. `GalleryListView.swift:144` does `let imageName = "Imported Image"` and passes it to `coordinator.createGalleryItem(name:imageData:)`, so every item gets the same label.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/GalleryListView.swift` (~line 144) — the hardcoded name at the import call site.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/ImagePickerView.swift` — the picker passes only `Data` via `onImageSelected: (Data) -> Void`; it doesn't surface a filename. macOS `NSOpenPanel` has the source `URL` (could derive a default name from `url.deletingPathExtension().lastPathComponent`); iOS `PHPickerResult.itemProvider.suggestedName` can supply one. Widening the closure to pass an optional suggested name would enable a good default.
- `GalleryItem.originalName` already stores the name and is rendered in `GalleryListView`/`GalleryDetailView`; the duplicate-prevention check (#0014) keys on content hash, not name, so naming is independent of de-dup.
- Two reasonable pieces: (1) a naming prompt/sheet at import time prefilled with the default; (2) inline rename of an existing item (e.g. a context-menu "Rename" → coordinator method that updates `originalName` and saves).
- PRD §1 lists "filename" as stored gallery metadata, so capturing the real source name is in scope.
