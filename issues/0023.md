## Description

Both the Mac and iPhone builds show a generic/blank app icon even though fully populated icon sets (`AppIcon-macOS`, `AppIcon-iOS`) exist in `PixelArtGallery/Assets.xcassets`.

Root cause is a build-setting precedence conflict: the target-level build settings in `PixelArtGallery.xcodeproj/project.pbxproj` set `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon` (Debug and Release), which overrides the per-SDK values in `Configuration/iOS.xcconfig` (`ASSETCATALOG_COMPILER_APPICON_NAME[sdk=iphone*] = AppIcon-iOS`) and `Configuration/macOS.xcconfig` (`[sdk=macos*] = AppIcon-macOS`). Target-level pbxproj settings always beat xcconfig values, so both platforms compile the `AppIcon` set ‚Äî and `AppIcon.appiconset` contains only a `Contents.json` with no image files, producing the generic icon.

## Steps to reproduce

1. Build and run the app on macOS ‚Äî the Dock/window icon is the generic app icon.
2. Build and run on the iOS Simulator ‚Äî the Home Screen icon for "Pixel Art Gallery" is blank/generic.

## Expected behavior

macOS builds use the `AppIcon-macOS` icon set; iOS builds use `AppIcon-iOS`.

## Actual behavior

Both builds compile the empty `AppIcon` set and show a generic icon.

## Notes

- Fix: remove the `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon` lines from the target's Debug and Release `buildSettings` in `project.pbxproj` so the per-SDK xcconfig values take effect. Consider deleting the empty `AppIcon.appiconset` so this can't regress silently.
- Verify with `xcodebuild -showBuildSettings` per destination (expect `AppIcon-macOS` / `AppIcon-iOS`) and by building both platforms.

## Root cause

The target's Debug and Release `buildSettings` in `PixelArtGallery.xcodeproj/project.pbxproj` both set `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon`. Target-level pbxproj settings take precedence over xcconfig values, so the per-SDK settings in `Configuration/iOS.xcconfig` (`AppIcon-iOS`) and `Configuration/macOS.xcconfig` (`AppIcon-macOS`) never applied, and both platforms compiled the `AppIcon.appiconset`, which contained only a `Contents.json` with no images.

## Fix

Removed the `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon;` line from both the Debug and Release target build configurations in `project.pbxproj`, letting the per-SDK xcconfig values take effect. Also deleted the empty `PixelArtGallery/Assets.xcassets/AppIcon.appiconset` so the dead icon set cannot be selected again. `AppIcon-iOS.appiconset` and `AppIcon-macOS.appiconset` were left untouched.

## Verification

- `xcodebuild -project PixelArtGallery.xcodeproj -scheme PixelArtGallery -destination 'platform=macOS' -showBuildSettings | grep ASSETCATALOG_COMPILER_APPICON_NAME` ‚Üí `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon-macOS`.
- Same with `-destination 'generic/platform=iOS'` ‚Üí `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon-iOS`.
- `xcodebuild -project PixelArtGallery.xcodeproj -scheme PixelArtGallery -destination 'platform=macOS' CODE_SIGNING_ALLOWED=NO build` ‚Üí BUILD SUCCEEDED.
- `xcodebuild -project PixelArtGallery.xcodeproj -scheme PixelArtGallery -destination 'platform=iOS Simulator,name=iPhone 17 Pro' CODE_SIGNING_ALLOWED=NO build` ‚Üí BUILD SUCCEEDED.
- Built macOS bundle contains `Contents/Resources/AppIcon-macOS.icns` and `Assets.car`, and Info.plist has `CFBundleIconFile` / `CFBundleIconName` = `AppIcon-macOS`.

## Files changed

- `PixelArtGallery.xcodeproj/project.pbxproj` ‚Äî removed the two target-level `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon;` lines (Debug and Release).
- `PixelArtGallery/Assets.xcassets/AppIcon.appiconset/Contents.json` ‚Äî deleted (empty icon set removed).
