# 0023 — App icon not applied on macOS or iOS despite icons existing in Assets

| | |
|---|---|
| **Status** | open |
| **Module** | App |
| **Platform** | All |
| **First seen** | 2026-07-06 |

## Description

Both the Mac and iPhone builds show a generic/blank app icon even though fully populated icon sets (`AppIcon-macOS`, `AppIcon-iOS`) exist in `PixelArtGallery/Assets.xcassets`.

Root cause is a build-setting precedence conflict: the target-level build settings in `PixelArtGallery.xcodeproj/project.pbxproj` set `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon` (Debug and Release), which overrides the per-SDK values in `Configuration/iOS.xcconfig` (`ASSETCATALOG_COMPILER_APPICON_NAME[sdk=iphone*] = AppIcon-iOS`) and `Configuration/macOS.xcconfig` (`[sdk=macos*] = AppIcon-macOS`). Target-level pbxproj settings always beat xcconfig values, so both platforms compile the `AppIcon` set — and `AppIcon.appiconset` contains only a `Contents.json` with no image files, producing the generic icon.

## Steps to reproduce

1. Build and run the app on macOS — the Dock/window icon is the generic app icon.
2. Build and run on the iOS Simulator — the Home Screen icon for "Pixel Art Gallery" is blank/generic.

## Expected behavior

macOS builds use the `AppIcon-macOS` icon set; iOS builds use `AppIcon-iOS`.

## Actual behavior

Both builds compile the empty `AppIcon` set and show a generic icon.

## Notes

- Fix: remove the `ASSETCATALOG_COMPILER_APPICON_NAME = AppIcon` lines from the target's Debug and Release `buildSettings` in `project.pbxproj` so the per-SDK xcconfig values take effect. Consider deleting the empty `AppIcon.appiconset` so this can't regress silently.
- Verify with `xcodebuild -showBuildSettings` per destination (expect `AppIcon-macOS` / `AppIcon-iOS`) and by building both platforms.
