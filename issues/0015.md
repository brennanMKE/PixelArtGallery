## Description

The PRD requires managing variants within a gallery item: view, edit dimensions, delete, or duplicate. Today `GalleryDetailView` only lists variants and navigates to them — there are no delete/duplicate/edit affordances. `GalleryCoordinator.deleteVariant(id:)` exists but mutates an in-memory relationship without a `ModelContext` and isn't surfaced in the UI.

## Expected behavior

From a gallery item the user can delete a variant (with confirmation), duplicate it, and edit its target dimensions (regenerating its pixel data). All operations persist through SwiftData and update the list immediately.
