# 0049 — Migrate all unit tests from XCTest to Swift Testing

| | |
|---|---|
| **Status** | open |
| **Module** | Testing |
| **Platform** | All |
| **First seen** | 2026-07-06 |

## Description

All unit test automation should use Swift Testing (`import Testing`, `@Test`, `@Suite`, `#expect`, `#require`), not XCTest. The package currently has 14 test files still on XCTest. Migrate them. UI tests are the exception and must remain XCTest — but there is no UI-test target today, so this migration covers the whole `PixelArtGalleryKitTests` suite.

## Long Description

`PixelationEngineTests.swift` was already converted to Swift Testing as part of #0048 and is the reference for the target style. The remaining files use `XCTestCase` subclasses, `XCTAssert*`, `XCTUnwrap`, and the `accuracy:` overloads — these map to `#expect`, `try #require`, and explicit tolerance checks (`abs(a - b) <= tol`). Async tests become `@Test func …() async throws`; expected-error tests become `#expect(throws:)`.

Files to migrate (all under `PixelArtGalleryKit/Tests/PixelArtGalleryKitTests/`):

- `AppLogTests.swift`
- `Export/PhotoLibrarySaverTests.swift`
- `Export/VariantExporterTests.swift`
- `ImageProcessing/PixelGridCGImageTests.swift`
- `Models/GallerySortOrderTests.swift`
- `Networking/DiscoveredFTDisplayTests.swift`
- `Networking/FTDisplayClientTests.swift`
- `Persistence/FileStorageManagerTests.swift`
- `Persistence/StorageFolderTests.swift`
- `UI/DisplayPreferredSelectionTests.swift`
- `UI/ManualDisplayInputTests.swift`
- `ViewModels/DisplayMergeTests.swift`
- `ViewModels/GalleryCoordinatorTests.swift`
- `ViewModels/PixelGridViewModelTests.swift`

## Steps to reproduce

1. `grep -rl "import XCTest" PixelArtGalleryKit/Tests/`
2. Observe 14 files still importing XCTest.

## Expected behavior

Every unit-test file uses Swift Testing. `grep -rl "import XCTest" PixelArtGalleryKit/Tests/` returns nothing (excluding any future UI-test target). `swift test` passes with the same or greater test count.

## Actual behavior

14 files subclass `XCTestCase` and use XCTest assertions.

## Notes

- Keep behavior identical — this is a mechanical framework migration, not a test-logic rewrite. Preserve each test's intent, messages, and any regression-issue references in the doc comments.
- Watch for `setUp`/`tearDown` (→ `init`/`deinit` or `@Suite` lifecycle), shared mutable state across tests (Swift Testing runs cases in parallel by default — may need `.serialized`), and `XCTestExpectation`-based async waits (→ native `async`/`await` or confirmations).
- Do NOT convert UI tests if a UI-test target is added later; those stay XCTest per project rule. See [[feedback_swift_testing]] equivalent project convention in `issues/Issues.md`.
