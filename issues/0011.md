## Description

Discovered and manually-entered FT displays need a place to live and be managed. There is currently no UI that lists stored `FlaschenTaschenDisplay` records, lets the user rename them, trigger a discovery scan, or remove stale entries. The `VariantDetailView` "Send to Display" picker is hardwired to `MockDisplay.samples` rather than real persisted displays.

## Expected behavior

A displays/registry screen lists all persisted FT displays with name, endpoint, and resolution; supports rename and delete; offers a "Scan" action (0009) and an "Add Manually" action (0010). Real displays from this registry replace the `MockDisplay` samples used in the send picker.

## Root cause

This was a feature gap, not a bug. Discovery (0009) and manual entry (0010) had
landed but had no UI home: nothing listed persisted `FlaschenTaschenDisplay`
records or let the user manage them, and `VariantDetailView`'s "Send to Display"
picker was still hardwired to `MockDisplay.samples` rather than reading the real
persisted registry.

## Fix

Added `DisplayRegistryView` under `UI/`: it `@Query`s all persisted displays
(name, endpoint, resolution), supports rename and delete, runs an mDNS scan, and
presents `ManualDisplayEntryView` for manual adds — reachable via a new
"Displays" toolbar entry / `GalleryRoute` nav destination in `GalleryListView`.
`GalleryCoordinator` gained `renameDisplay`, `deleteDisplay`, `scanForDisplays`
(streams `FTDiscoveryService` then merges), and `mergeDiscoveredDisplays`, which
applies a new pure `DisplayMergePlan`. The plan de-dupes by a normalized
`host:port` key (host lowercased): matched endpoints update an existing record in
place (refreshing dimensions/name, setting `source = "mdns"`), unmatched ones
insert new `mdns` records, and duplicate discoveries within a single scan
collapse (last wins). Finally, `VariantDetailView`'s send picker now `@Query`s
real `FlaschenTaschenDisplay` records (selection by `id`, with an empty-state
hint); `MockDisplay` was removed from `GalleryModels.swift`. The network send is
left stubbed for issue 0012 but is now driven by the real selected display.
