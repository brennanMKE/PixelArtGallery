## Description

Expand the bundled built-in sprite set from 11 to 22. Add new sprite definitions to `scripts/sprites.py`, regenerate the PNGs, and wire the new names into the app's preloaded (built-in) sprites so they appear in the gallery's Sprites section on launch. Also switch the generated background from near-black to pure black so it's transparent on Flaschen Taschen layers.

## Long Description

The app ships built-in sprites (#0074, #0075), each a 45×35 PNG generated by `scripts/sprites.py` and bundled under `PixelArtGalleryKit/Sources/PixelArtGalleryKit/Resources/DefaultSprites/`. The `GalleryCoordinator.builtInSpriteNames` manifest drives the always-present/auto-restoring reconcile; the resource file for each name must exist in `DefaultSprites/` or the reconcile logs a miss and skips it.

This issue adds eleven more sprites, in two groups:

- **Classic arcade icons (7):** `ufo`, `squid`, `octopus` (the Space Invaders mystery ship plus the other two aliens, completing the trio with the existing `invader` crab), `barrel` (Donkey Kong), `bomb` (Bomberman), `cherry` (Pac-Man fruit), and `key` (dungeon/adventure).
- **Mario-universe characters (4):** `mario`, `luigi` (a red→green palette swap of Mario), `princess` (Peach), and `bowser`.

It also makes two whole-set changes: the generator's background is now pure black (`#000000`) instead of `(12, 12, 20)`, so the padding and negative space around every sprite is **transparent** on FT and lower layers show through; and three palette colors were added — `n` brown (hair/mustache/shoes/shells), `i` pink (Peach's dress), `t` tan (Bowser's snout/belly).

## Fix

1. **`scripts/sprites.py`**
   - Added the 11 new sprite grids to the `sprites` dict. `luigi` is generated from `mario`'s grid with `r`→`g` so the two stay shape-identical.
   - Changed the `BG` constant from `(12, 12, 20)` to `(0, 0, 0)` (transparent on FT).
   - Added palette entries `n`/`i`/`t` for the character sprites.
   - **bomb** uses dark grey (`d`) for the body, not black (`k`): pure black is transparent on FT, so a black body would drop out; dark grey stays visible while still reading as a bomb.
   - **barrel** redesigned from the original vertical-pinstripe look (which read as a jail cell) to a rounded cask: orange staves, two horizontal dark hoop bands, and a pair of sheen stripes.
2. **PNGs regenerated** for the whole set (the `BG` change touches every sprite) and the new ones copied into `DefaultSprites/` (22 PNGs total).
3. **`GalleryCoordinator.builtInSpriteNames`** extended to the full 22 names (kept alphabetical). The always-present reconcile inserts the new sprites and, because every existing sprite's content hash changed with the `BG` swap, updates all prior sprites in place via the #0075 update path — no migration.
4. **`BuiltInSpritesTests`** count assertions derive from `GalleryCoordinator.builtInSpriteNames.count` rather than hardcoded numbers, so the manifest is the single source of truth and further additions don't require re-editing every assertion.
