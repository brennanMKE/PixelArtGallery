## Description

The `PixelationEngine` currently downsamples with nearest-neighbor sampling (`interpolationQuality = .none`), placed there to get the build compiling. The PRD specifies the downsampling algorithm as **bilinear interpolation with a light blur** (matching the PixelArtConverter reference) for color accuracy.

## Expected behavior

Downsampling averages source pixels (bilinear/area interpolation) with an optional light pre-blur so each output cell reflects the true average color of its source region rather than a single sampled pixel. Output remains RGBA8888 `PixelGrid` at the requested dimensions. Color accuracy is preserved per the PRD success criteria.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ImageProcessing/PixelationEngine.swift` — `pixelate(_:targetWidth:targetHeight:)`; switch interpolation quality and/or add a Core Image / vImage blur+resample pass.
- PRD §5 "Color Sampling & Display" and §"Image Processing".
- Worth adding unit tests asserting averaged output for a known gradient.

## Root cause

`PixelationEngine.pixelate(_:targetWidth:targetHeight:)` set `context.interpolationQuality = .none` (nearest-neighbor) as a placeholder to get the build compiling. On downscale, nearest-neighbor picks a single source pixel per output cell and discards the rest of the region, so the resulting cell color does not represent the average of the area it covers — a color-accuracy regression against the PRD.

## Fix

Switched the downsample draw to `context.interpolationQuality = .high` and explicitly enabled `context.setShouldAntialias(true)`. At high quality Core Graphics area-averages the source pixels that fall under each destination cell (bilinear/area resampling with light blur), so every output cell reflects the true average color of its source region. The public API, the RGBA8888 premultiplied-last buffer, and the `PixelGrid` output shape are all unchanged; only the resampling algorithm changed. Code landed in 7379404.

## Gotchas

- The checkerboard test deliberately asserts the result is mid-gray AND explicitly rejects the pure-black (0) / pure-white (255) outcomes a nearest-neighbor implementation would produce, so the test fails if someone reverts to `.none`. A tolerance is used to absorb rounding and color-space conversion.
