## Description

The app feels flat. The sibling project `../PixelArtConverter` has a lively, fun aesthetic we want here: a vibrant 4-color pixel palette, a full-bleed randomized pixel background (`BackgroundPixelsView`), an animated 2×2 pixel block that recolors every 0.5s (`AnimatedPixelsView`), translucent "frosted" cards that let the background show through, and a pixel-art app icon.

## Expected behavior

- A subtle animated/colorful pixel background sits behind the app on the main screens (light/dark aware).
- The app uses the PixelArtConverter palette (gold #F7B900, magenta #CE06A8, green #09CE07, red #CE0C05) as its accent, and cards frost over the background.
- The empty gallery state features the animated pixel block so first launch feels alive.
- The app has the PixelArtConverter-style app icon on iOS and macOS.

## Notes

- Source: `../PixelArtConverter/PixelArtConverterKit/Sources/PixelArtConverterKit/Views/{BackgroundPixelsView,AnimatedPixelsView}.swift` and `Utility/Colors.swift`. Composition pattern (from `MainView`): `ZStack { BackgroundPixelsView().ignoresSafeArea(); content }` with `.toolbarBackground(.hidden)`; `AnimatedPixelsView` used as a hero in `RootView`.
- App icon: copy `../PixelArtConverter/PixelArtConverter/Assets.xcassets/AppIcon-iOS.appiconset` + `AppIcon-macOS.appiconset` into this app's `Assets.xcassets`. The gallery's `Configuration/{iOS,macOS}.xcconfig` already set `ASSETCATALOG_COMPILER_APPICON_NAME` to `AppIcon-iOS`/`AppIcon-macOS`, which currently don't exist (only a stray `AppIcon.appiconset`) — so this also fixes a missing icon.
- Integration touchpoints in this app: `ContentView` (ZStack + background + `.tint`), the List-based screens need `.scrollContentBackground(.hidden)` to show the background through, `Theme.card()` becomes translucent (material), and `EmptyStateView` gains the animated block. Builds on the design system from #0019. Same author owns both projects, so porting is fine.

## Root cause

Feature gap, not a bug: the app was visually flat, with no color or motion. The sibling `PixelArtConverter` project already had the desired lively aesthetic to port.

## Fix

Ported the aesthetic (see the code commit body for specifics): a vibrant palette (`Palette.swift`), a full-bleed randomized pixel wallpaper (`BackgroundPixelsView`) placed *inside* the NavigationStack, an animated 2×2 block (`AnimatedPixelsView`) used as the empty-state hero, frosted `.regularMaterial` cards, `.scrollContentBackground(.hidden)` + hidden toolbar backgrounds so the wallpaper shows through, a gold app-wide accent tint, and the PixelArtConverter app-icon sets.

Key gotcha: the background must live **inside** the `NavigationStack` (as a back layer of a `ZStack` around the content). Placed *outside/behind* the NavigationStack, its opaque system background hid the wallpaper entirely (first screenshot was blank white) — matching PixelArtConverter's `MainView` composition fixed it.
