## Description

Model/coordinator foundation for the popover flow ([#0065](0065.md)). Add: (1) a **transient fitted-preview** value type — the fitted (aspect-preserving) pixel grid + fit dimensions + centering offset + source display id — built for an (item, display) pair without persisting anything; (2) an in-memory **cache** so re-selecting a display is instant; (3) a **`saveVariant(from:)`** path that persists a transient preview as a `Variant` (so it appears in the saved-variants list) only when the user saves/edits, with dedup; and (4) **last-used-display persistence** so the popover can default its dropdown. Reuses [#0062](0062.md)'s `AspectFit`; reworks [#0063](0063.md) so fitted variants are transient by default rather than always persisted.

## Long Description

Today `GalleryCoordinator.createFittedVariant(for:display:)` (from #0063) always inserts + saves a `Variant`. The popover flow needs a fitted result the moment a display is selected, but it must NOT accumulate saved variants for every glance — only an explicit save persists one. So the fitted result becomes a transient value (grid + fit dims + centering offset + display id), computed via `AspectFit.fit(...)` and the existing pixelation, cached in memory, and sendable directly. Persistence is opt-in via a save action.

## Plan

### Context (verified against the code)

- `GalleryCoordinator.createFittedVariant(for:display:)` (`GalleryCoordinator.swift:302`) always persists: it computes `AspectFit.fit` from `item.originalWidth/originalHeight` (decode fallback when ≤ 0), dedups against `item.variants`, then calls `createVariant(...)` which inserts + saves. Its **only production call site** is `DisplaySendPickerView.send(to:)` (`UI/DisplaySendPickerView.swift:195`); the other references are the three tests in `GalleryCoordinatorTests.swift` (lines 382/410/436). `DisplaySendPickerView` is slated for retirement in #0068, so this issue should keep it compiling with unchanged behavior rather than rework it.
- `PixelationEngine.processFitting(imageData:displayWidth:displayHeight:)` (`PixelationEngine.swift:59`) already returns `(grid: PixelGrid, placement: AspectFit.Placement)` — fit-sized grid, letterbox NOT baked in, centering in `placement.offsetX/offsetY`. It is tested (`PixelationEngineTests.swift:117-156`, including EXIF orientation) but has zero production callers (the #0062 dead-code concern).
- `AspectFit.Placement` already carries `{width, height, offsetX, offsetY}` and is `nonisolated`/`Sendable`; `fit(...)` calls `centeringOffset(...)` internally.
- Important subtlety: `createGalleryItem` records `originalWidth/originalHeight` from a **raw** `CGImageSourceCreateImageAtIndex` decode (no EXIF transform), while the engine pixelates via `decodeUpright` (EXIF-transformed). For an EXIF-rotated photo the stored dims are the un-rotated buffer's — computing fit from stored dims can produce a wrong-aspect grid. This drives the pixelation choice below.
- The package default-isolates the main target to `@MainActor` (`Package.swift`: `.defaultIsolation(MainActor.self)`), so any pure value type used from nonisolated contexts must be explicitly `nonisolated`. The test harness (`GalleryCoordinatorTests.swift`) is Swift Testing: `@MainActor @Suite final class` with a per-test `tempDirectory`, `makeCoordinator()` (temp `FileStorageManager`), `makeContext()` (in-memory `ModelContainer`), `makeItem(...)` (imports a 16×16 PNG from `makePNGData`).
- Precedent for lightweight persistence: `@AppStorage("gallerySortOrder")` in `GalleryListView.swift:44` — plain UserDefaults, no SwiftData schema involvement.

### 1. New file: `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ViewModels/FittedPreview.swift`

(Under `ViewModels/`, not `Models/` — per `issues/Issues.md` module conventions, `Models` means SwiftData `@Model` types; this is deliberately *not* persisted.)

```swift
/// Transient fitted preview of a gallery item for one FT display. Never persisted.
nonisolated struct FittedPreview: Equatable, Sendable {
    let itemID: UUID        // identifies the source GalleryItem for saveVariant + cache key
    let displayID: UUID     // becomes Variant.associatedDisplayId on save; routes the send
    let width: Int          // FIT width (aspect-preserving), == grid width, NOT display width
    let height: Int         // FIT height, == grid height
    let pixelGridData: Data // RGBA8888 (width*height*4), same encoding Variant persists
    let offsetX: Int        // centering offset on the display (AspectFit), x
    let offsetY: Int        // centering offset, y
}
```

Field justification: `itemID`/`displayID` are what `saveVariant(from:)` needs to re-locate the item and tag the variant, and half the cache key; `width`/`height` are the fit dims the SwiftUI preview renders at and the FT send transmits; `pixelGridData` is RGBA8888 so saving is a byte-copy into `Variant.pixelGridData` with no re-pixelation, and the send/preview paths consume the exact format they already handle; `offsetX`/`offsetY` are the centering placement the send adds to the display's stored `offsetX/offsetY` (#0056) and the preview uses for positioning — carried on the value so #0067 never re-derives them. `Equatable` makes cache-hit tests a one-line `#expect`; `nonisolated ... Sendable` because the package default-isolates to `@MainActor` and the value crosses into nonisolated test/async contexts (same reasoning as `AspectFit`, which bit #0057/#0060).

Also in this file, the cache key:

```swift
nonisolated struct FittedPreviewCacheKey: Hashable, Sendable {
    let itemID: UUID
    let displayID: UUID
    let displayWidth: Int
    let displayHeight: Int
}
```

Display dims are in the key for the same reason #0063's dedup used them: a display's geometry can change in place (mDNS re-scan, `updateDisplay`), and a stale-geometry preview must miss, not hit.

### 2. `GalleryCoordinator`: `fittedPreview(for:display:)` — compute, cache, never persist

```swift
func fittedPreview(for item: GalleryItem, display: FlaschenTaschenDisplay) async throws -> FittedPreview
```

- Cache lookup first, keyed by `FittedPreviewCacheKey(itemID: item.id, displayID: display.id, displayWidth: display.displayWidth, displayHeight: display.displayHeight)` — note the key needs no fit math and no I/O, so a hit returns instantly.
- On miss: load original bytes via `fileStorageManager()` (throw `originalImageMissing` when nil, as `createVariant` does), then call **`PixelationEngine.processFitting(imageData:displayWidth:displayHeight:)`** — one call yields the fit-sized grid *and* the `AspectFit.Placement` (fit dims + centering; `centeringOffset` is applied inside `AspectFit.fit`).
- Build `FittedPreview(itemID: item.id, displayID: display.id, width: placement.width, height: placement.height, pixelGridData: grid.toRGBA8888(), offsetX: placement.offsetX, offsetY: placement.offsetY)`, store in the cache, return. **No `modelContext` guard, no insert, no save** — the method is read-only with respect to SwiftData.
- Error handling/logging mirror the existing methods: set `currentError`, log via `AppLog.variant`, rethrow.

**Why `processFitting` over `process` at pre-computed fit dims (this retires the #0062 dead-code concern):** (a) it computes fit from the **decoded, EXIF-upright** dimensions, so it cannot disagree with the pixels the way stored `originalWidth/originalHeight` can (see Context — stored dims come from a non-EXIF-transformed decode); (b) #0063 chose `process` only because its dedup needed fit dims *before* pixelation, but the preview path's cache key uses display dims, so nothing here needs fit dims early; (c) on a cache miss the bytes must be loaded and decoded anyway, so the stored-dims fast path buys nothing — the "decode-bytes fallback when ≤ 0" becomes moot because `processFitting` always derives dims from the decode. Its EXIF and invalid-dimension behavior is already covered by `PixelationEngineTests`. When implementing, also delete/soften the now-stale "processFitting is not needed here" paragraph in the old `createFittedVariant` doc comment (`GalleryCoordinator.swift:290-295`).

### 3. `GalleryCoordinator`: `saveVariant(from:)` — the explicit persistence path

```swift
@discardableResult
func saveVariant(from preview: FittedPreview) throws -> Variant
```

- Guard `modelContext` (throw `.missingModelContext`). Fetch the `GalleryItem` by `#Predicate { $0.id == preview.itemID }` (fetchLimit 1); if absent throw a new `GalleryCoordinatorError.galleryItemNotFound(UUID)` case (extend the existing `nonisolated enum` + `errorDescription`).
- **Dedup** (moved verbatim from #0063's `createFittedVariant`): filter `item.variants` for `associatedDisplayId == preview.displayID && targetWidth == preview.width && targetHeight == preview.height`; if any, return the one with the latest `createdDate` — do not touch its pixel data (preserves the "never stomp user edits" rationale documented at `GalleryCoordinator.swift:283-288`).
- Otherwise: `Variant(targetWidth: preview.width, targetHeight: preview.height, pixelGridData: preview.pixelGridData, associatedDisplayId: preview.displayID)`, set `galleryItem`, append to `item.variants`, `insert`, `save`. Synchronous `throws` (no engine work — the grid is already computed), matching `duplicateVariant`'s shape. Log + `currentError` on failure as elsewhere.

### 5. In-memory cache

On `GalleryCoordinator`:

```swift
@ObservationIgnored private var previewCache: [FittedPreviewCacheKey: FittedPreview] = [:]
```

- `@ObservationIgnored` (implementation detail, must not invalidate views — same as `modelContext`/`fileStorage`). The coordinator is `@MainActor` via the package default isolation, so plain dictionary access is safe.
- Not persisted anywhere; memory cost is negligible (a 45×35 preview is ~6 KB), so no size cap and no memory-pressure hook — keep it simple.
- Invalidation: a private `invalidatePreviews(forItemID: UUID)` that strips matching keys, called from `deleteGalleryItem(_:)`. Display-geometry changes need no explicit invalidation because the dims are in the key (stale entries are unreachable and tiny). Item *content* never changes after import (originals are immutable), so no other hook is needed.

### 6. Last-used-display persistence

**Choice: plain `UserDefaults` accessed by the coordinator (key `"lastUsedFTDisplayID"`, String UUID), with the `UserDefaults` instance injected via init — not `@AppStorage`.** Justification: `@AppStorage` is a SwiftUI dynamic property that only functions inside `View`s; hosted on an `@Observable` class it neither observes nor publishes correctly, so it cannot live on `GalleryCoordinator`. Putting the logic in the coordinator (rather than an `@AppStorage` in the popover view) makes the default-selection rule unit-testable without SwiftUI and shared across the iOS/macOS popovers. No SwiftData schema change. Should #0067 ever want live observation of the raw value in a view, `@AppStorage("lastUsedFTDisplayID")` reads the same key/store.

- Init becomes `init(fileStorage: FileStorageManager? = nil, defaults: UserDefaults = .standard)`; store as `@ObservationIgnored private let defaults`. Static key constant, e.g. `static let lastUsedDisplayIDKey = "lastUsedFTDisplayID"`.
- `func rememberLastUsedDisplay(_ display: FlaschenTaschenDisplay)` — writes `display.id.uuidString`. #0067 calls this on dropdown selection and on send.
- `func resolveLastUsedDisplay(among candidates: [FlaschenTaschenDisplay]) -> FlaschenTaschenDisplay?` — reads the stored string, parses UUID, returns the matching candidate; when absent/unparseable/no longer present, falls back to the candidate with `source == FlaschenTaschenDisplay.defaultSource`, else `candidates.first`, else `nil` (mirrors the fallback ladder of `preferredSelection`, `FlaschenTaschenDisplay.swift:158-177`). Candidates come from the view's `@Query`, so the coordinator does no fetching here. This is the default-selection entry point #0067 consumes.

## Root cause

Not a bug fix — a feature gap. `GalleryCoordinator.createFittedVariant(for:display:)` (#0063) always inserted + saved a `Variant` the moment a display was fitted, so the popover flow ([#0067](0067.md)) had no way to show a live fitted preview as the user browses displays without silently accumulating a saved variant per glance. There was also no persisted "last selected FT display" the popover could default to, and `PixelationEngine.processFitting(imageData:displayWidth:displayHeight:)` had zero production callers (the #0062 dead-code concern) because #0063 needed fit dimensions computed *before* pixelation (for its dedup check) and so used `process` at pre-computed dims instead.

## Fix

- Added `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ViewModels/FittedPreview.swift`: `nonisolated struct FittedPreview: Equatable, Sendable { itemID, displayID, width, height, pixelGridData, offsetX, offsetY }` and `nonisolated struct FittedPreviewCacheKey: Hashable, Sendable { itemID, displayID, displayWidth, displayHeight }` — both `nonisolated` because the package default-isolates to `@MainActor` and these values cross into nonisolated test/async contexts (the #0057/#0060 pattern).
- Added `GalleryCoordinator.fittedPreview(for:display:) async throws -> FittedPreview`: checks the in-memory cache first (`@ObservationIgnored private var previewCache: [FittedPreviewCacheKey: FittedPreview]`), and on a miss loads the original bytes and calls `PixelationEngine.processFitting(imageData:displayWidth:displayHeight:)` — one call yields both the fit-sized grid and the `AspectFit.Placement` (fit dims + centering). No `modelContext` guard, no insert, no save — purely read-only with respect to SwiftData. **This gives `processFitting` its first production caller, retiring the #0062 dead-code concern.** `processFitting` was chosen over `process` at pre-computed dims because it derives the fit from the decoded, EXIF-upright image, so it can't disagree with the actual pixels the way the item's stored `originalWidth`/`originalHeight` (from a non-EXIF-transformed decode at import time) can — and because the preview's cache key uses display dimensions rather than fit dimensions, nothing needs fit dims computed early the way #0063's dedup did.
- Added `GalleryCoordinator.saveVariant(from preview: FittedPreview) throws -> Variant` (`@discardableResult`): guards `modelContext`, re-fetches the `GalleryItem` by `preview.itemID` (throwing a new `GalleryCoordinatorError.galleryItemNotFound(UUID)` case if it's gone), dedups against `item.variants` on `associatedDisplayId == preview.displayID && targetWidth == preview.width && targetHeight == preview.height` (moved verbatim from #0063's dedup), and otherwise builds a `Variant` directly from the preview's already-computed `pixelGridData` (no re-pixelation), attaches it, inserts, and saves.
- Refactored `createFittedVariant(for:display:)` into a thin wrapper — `fittedPreview(...)` then `saveVariant(from:)` — so `DisplaySendPickerView.send(to:)`, its sole remaining production call site, keeps compiling with identical observable (always-persist, deduped) behavior until #0068 retires that view and this wrapper together.
- Added cache invalidation: `invalidatePreviews(forItemID:)` strips matching keys, called from `deleteGalleryItem(_:)`. No hook is needed for display-geometry changes since display dimensions are already part of the cache key — stale entries just become unreachable.
- Added last-used-display persistence as plain injected `UserDefaults` (key `GalleryCoordinator.lastUsedDisplayIDKey = "lastUsedFTDisplayID"`, `String` UUID) rather than `@AppStorage`, because `@AppStorage` is a SwiftUI dynamic property that doesn't function correctly hosted on an `@Observable` class. `init(fileStorage:defaults:)` now takes an optional `UserDefaults` (default `.standard`) so tests can inject an isolated suite. `rememberLastUsedDisplay(_:)` writes the id; `resolveLastUsedDisplay(among:)` reads it back, falling back to the `defaultSource` candidate, then the first candidate, then `nil` — mirroring `FlaschenTaschenDisplay.preferredSelection`'s fallback ladder.
- No deviations from the plan.

Design points (planner to finalize):
- **Transient preview type**: a `nonisolated`/`Sendable` value type, e.g. `struct FittedPreview { let itemID: UUID; let displayID: UUID; let width: Int; let height: Int; let pixelGridData: Data; let offsetX: Int; let offsetY: Int }` (offset = `AspectFit` centering; width/height = fit dims). Include what the send path and the SwiftUI preview need. Consider deriving the preview grid via the existing `process(imageData:targetWidth:targetHeight:)` at fit dims (aspect-correct by construction) or `processFitting(...)` — pick one and justify; if `processFitting` is finally used here, note that it retires the dead-code concern from #0062.
- **Coordinator API**: `func fittedPreview(for item: GalleryItem, display: FlaschenTaschenDisplay) async throws -> FittedPreview` (computes fit dims via `AspectFit.fit` from `item.originalWidth/originalHeight` with a decode fallback, pixelates aspect-fit, returns transient — NO insert/save); and `func saveVariant(from preview: FittedPreview) throws -> Variant` (persists as a `Variant` with `associatedDisplayId = displayID`, dedup against an existing matching saved variant, appears in `item.variants`). Refactor #0063's `createFittedVariant` to build on these (or replace it) so the always-persist behavior no longer fires on mere selection — update its call sites accordingly.
- **In-memory cache**: cache the transient preview keyed by (itemID, displayID, displayWidth, displayHeight) so re-selecting is instant and doesn't re-pixelate; the cache is not persisted and can be cleared on memory pressure / item change. Keep it simple.
- **Last-used-display persistence**: persist the last-used display id (e.g. `@AppStorage("lastUsedFTDisplayID")` as a String, or a small stored setting) with a helper to resolve it to a current `FlaschenTaschenDisplay` (falling back to the first available if it's gone). Update it whenever the user selects a display / sends. Provide the default-selection logic the popover ([#0067](0067.md)) will consume.
- **No unnecessary schema change**: prefer not to add `Variant` fields; if last-used needs storage, `@AppStorage`/UserDefaults avoids touching the SwiftData schema.

Testing (Swift Testing, in-memory `ModelContainer` + temp `FileStorageManager`, mirroring `GalleryCoordinatorTests`): `fittedPreview` returns fit dims (not display dims) for an aspect-mismatched pair, correct centering offset (via `AspectFit`), and does **NOT** insert a `Variant` (assert `item.variants.count` unchanged, no fetchable Variant); the cache returns an equal preview on re-invocation without a second pixelation (observe via a spy or by identity of cached data); `saveVariant(from:)` persists exactly one `Variant` with the right dims/offset/association and dedups on repeat; last-used-display round-trips (set → resolve returns the same display; resolves to a fallback when the stored id is absent). Note that the pure fit/centering correctness is already covered by #0062.
