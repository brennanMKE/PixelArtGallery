# 0008 — Support exporting variants to the Photos library on iOS

| | |
|---|---|
| **Status** | open |
| **Module** | Export |
| **Platform** | iOS |
| **First seen** | 2026-06-25 |

## Description

The PRD lists "Save to file or Photos app (iOS only)" as an export destination. On iOS there is no file-system save panel; `ExportPickerView.showSavePanel()` just sets `isShowingSavePanel = true` with a `// TODO` and goes nowhere. There is no path to save an exported image into the Photos library, even though the Info.plist permission key is already configured.

## Expected behavior

On iOS, the user can export a variant's rendered image directly into the Photos library (and/or via the Files document picker). The save flow requests authorization and reports success/failure. The `NSPhotoLibraryAddUsageDescription` string already present in `Configuration/iOS.xcconfig` is used.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/ExportPickerView.swift` — the `#else` (iOS) branch of `showSavePanel()`.
- Use `PHPhotoLibrary` to add the exported PNG/HEIC; pair with the real exporter from 0007.
- `Configuration/iOS.xcconfig` — `INFOPLIST_KEY_NSPhotoLibraryAddUsageDescription` already set.
- Depends on 0007.
