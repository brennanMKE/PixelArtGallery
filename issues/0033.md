## Description

The main Gallery screen renders imported images as a `List` of rows (60×60 thumbnail plus text), which leaves a lot of empty space and doesn't look like a gallery — especially on the Mac where rows stretch across a wide window. The original vision is a proper gallery: a grid of image thumbnails that adapts well to the width of the scene, filling the space with art.

## Expected behavior

- The gallery shows items in a grid (e.g. `LazyVGrid` with adaptive columns) that reflows with the window/scene width: more columns on a wide Mac window or iPad, fewer on iPhone.
- Each cell leads with a generously sized thumbnail (square, rounded, consistent with the app's card treatment), with the item's name and variant count as compact captions.
- Existing behavior is preserved: tapping/clicking a cell navigates to the item's detail; the context menu still offers Rename and Delete (with the #0029 confirmation); the empty state and import flow are unchanged.

## Actual behavior

A single-column list with small thumbnails and wide empty rows.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/GalleryListView.swift` — the `List`/`ForEach` block, including the context menu (Rename/Delete), delete confirmation, and iOS swipe `.onDelete` (swipe doesn't apply to grid cells; the confirmed context-menu Delete from #0029 covers deletion on both platforms).
- `StoredImageView` already takes a `maxPixelSize` — bump it appropriately for larger grid thumbnails (currently 180 for the 60pt row).
- Keep `BackgroundPixelsView` visible behind the grid (the list currently uses `.scrollContentBackground(.hidden)`); a `ScrollView` + `LazyVGrid` shows it naturally.

## Root cause

The gallery section of `GalleryListView` was built as a `List` of `HStack` rows (60×60 thumbnail + three lines of text), which is a fine index but not a gallery: on a wide Mac window each row stretched the full width with mostly empty space, and thumbnails stayed tiny regardless of available room. Nothing adapted to scene width.

## Fix

Replaced the `List`/`ForEach` with `ScrollView` + `LazyVGrid` using a single `GridItem(.adaptive(minimum: 150, maximum: 220), spacing: Theme.Spacing.m)` — adaptive sizing lets SwiftUI derive the column count from the scene width (150pt minimum keeps cells legible on iPhone; 220pt maximum stops cells from ballooning on very wide Mac windows), with `Theme.Spacing.m` gutters and `Theme.Spacing.l` outer padding. Each cell (new private `galleryCell(for:)`) is a square thumbnail — `Color.clear.aspectRatio(1, contentMode: .fit)` defines the square footprint, `StoredImageView` (default `.fill` content mode, `maxPixelSize` bumped 180 → 540 so ~200pt cells stay sharp on 2x/3x displays) fills it via `.overlay`, clipped to `RoundedRectangle(cornerRadius: Theme.Radius.card)` over a `.quaternary` fill — followed by the name (`.headline`, `lineLimit(1)`) and the variant count caption (existing pluralization). Cells stay wrapped in `NavigationLink(value: item)` with `.buttonStyle(.plain)` so text doesn't tint, and keep the Rename/Delete context menu with the #0029 confirmation dialog unchanged. The iOS swipe `.onDelete` was removed (doesn't apply to grid cells; confirmed context-menu Delete covers both platforms), and `.scrollContentBackground(.hidden)` was dropped since `ScrollView` is transparent, so `BackgroundPixelsView` shows through naturally. Empty state, alerts, toolbar, pickers/sheets, and coordinator wiring untouched.

## Gotchas

- `FileStorageManager` writes to the real `~/Library/Application Support/PixelArtGallery/Images` when run outside the app sandbox, so the harness tracked each imported `originalImagePath` and deleted those files on exit. (Separately observed, pre-existing: `FileStorageManagerTests` leaves small fixture `.dat` files in that directory — e.g. `testSaveAndLoadImageData` never deletes its file. Not addressed here.)
- The square-thumbnail trick matters: putting `.aspectRatio(1, contentMode: .fit)` on the image itself would letterbox it; sizing a `Color.clear` square and overlaying the `.fill`-mode `StoredImageView` (then clipping) crops any aspect ratio to a clean square cell.
