# 0001 — Present the gallery UI from the app entry point instead of the placeholder ContentView

## Description

The app launches to the default `ContentView` ("Hello, world!") and never shows the gallery. `GalleryListView` (the real root of the app) and the other Kit views are `internal`, so the app target can't present them. Nothing the user builds is reachable until this is wired up.

## Expected behavior

Launching the app shows `GalleryListView` as the root, inside the `WindowGroup` that already injects the `ModelContainer`. The placeholder `ContentView` is either replaced or repurposed to host the gallery.

## Notes

- `PixelArtGallery/ContentView.swift` — current placeholder.
- `PixelArtGallery/PixelArtGalleryApp.swift` — `WindowGroup { ContentView() }.modelContainer(...)`.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/GalleryListView.swift` — make this (and any types it exposes) `public`, including a public initializer.
- Depends on nothing else, but is a prerequisite for exercising every other UI issue end to end.

## Root cause

`GalleryListView` (the intended app root) and its synthesized initializer were `internal` to the `PixelArtGalleryKit` package, so the `PixelArtGallery` app target could not reference or construct them. The app's `WindowGroup` therefore presented the default placeholder `ContentView` ("Hello, world!"), and the real gallery was never reachable.

## Fix

Made `GalleryListView` a `public struct` and added an explicit `public init() {}` (plus a `public` `body`) so the app target can construct it. Repurposed `ContentView` to host `GalleryListView()`, importing `PixelArtGalleryKit`. The gallery is now the app root inside the existing `WindowGroup`, which already injects the `ModelContainer`. `GalleryCoordinator` stayed internal since `GalleryListView` instantiates it itself.

## Verification

- `xcodebuild -project PixelArtGallery.xcodeproj -scheme PixelArtGallery -destination 'platform=macOS' build` — **BUILD SUCCEEDED**.
- `xcodebuild -project PixelArtGallery.xcodeproj -scheme PixelArtGallery -destination 'platform=iOS Simulator,name=iPhone 17 Pro' build` — **BUILD SUCCEEDED**.

This is UI wiring with no new testable logic, so the two platform builds are the verification.

## Files changed

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/GalleryListView.swift` — made the struct, its `init()`, and `body` `public` so the app target can present it.
- `PixelArtGallery/ContentView.swift` — imported `PixelArtGalleryKit` and replaced the placeholder body with `GalleryListView()`.
