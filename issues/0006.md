## Description

The PRD calls for an interactive grid where the user can hover or tap a pixel to view its color value. `PixelGridViewModel` already models this (`selectedPixel`, `selectPixel(x:y:)`, `clearSelection()`), but `PixelGridView` never wires any gesture to it and never displays the selected color, so the feature is dead code.

## Expected behavior

Tapping (iOS) or hovering (macOS) a cell in the Canvas selects that pixel, highlights it, and shows its coordinate and RGBA value. Selection clears appropriately. Works at all zoom levels.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/PixelGridView.swift` — add a tap/hover gesture over the `Canvas`, map the location through `pixelSize * zoomLevel` to grid coordinates, call `viewModel.selectPixel`, and render `viewModel.pixelColor(x:y:)`.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ViewModels/PixelGridViewModel.swift` — selection API already present.
- PRD §5 "Interactive Grid".

## Root cause

`PixelGridViewModel` exposed a complete selection API (`selectedPixel`, `selectPixel(x:y:)`, `clearSelection()`, `pixelColor(x:y:)`), but `PixelGridView` rendered the `Canvas` with no gesture attached and no UI to surface the selection. Nothing ever called the selection methods, so the inspection feature was unreachable dead code.

## Fix

Added a testable `gridCoordinate(at:)` to the view model that maps a Canvas-space point through `pixelSize * zoomLevel` into integer grid coordinates, returning `nil` when the point is outside the grid. Layered a `selectPixel(at:)` convenience on top that selects the hit pixel or clears the selection. In `PixelGridView`, wired a platform-conditional gesture over the `Canvas` (`DragGesture(minimumDistance: 0)` on iOS, `onContinuousHover` on macOS), drew a yellow highlight stroke on the selected cell inside `drawPixelGrid`, and added a readout row above the grid showing the coordinate, a color swatch, and the RGBA value from `pixelColor(x:y:)`.
