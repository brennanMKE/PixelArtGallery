# 0029 — Gallery items cannot be deleted on macOS

## Description

The gallery list's only delete affordance is `List.onDelete`, which is a swipe gesture on iOS. On macOS there is no visible way to delete a gallery item: the row's context menu offers only Rename, and no toolbar or menu command deletes. Users can import images but never remove them on the Mac.

## Steps to reproduce

1. Run the macOS app with at least one imported image.
2. Right-click the item — the context menu has only Rename. There is no other delete affordance.

## Expected behavior

A Delete action on the row's context menu (both platforms — it also helps discoverability on iOS), with a confirmation since deletion also removes the item's variants and stored image files.

## Actual behavior

No way to delete a gallery item on macOS.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/GalleryListView.swift` — `.onDelete` at the `ForEach`, context menu with Rename only.
- Match the destructive-confirmation pattern used for variants in `GalleryDetailView` (confirmationDialog naming what is deleted).

## Root cause

The only delete affordance was `List.onDelete`, which maps to swipe-to-delete on iOS and has no equivalent gesture on macOS. The row's context menu — the natural macOS delete surface — offered only Rename, so nothing on the Mac ever called `GalleryCoordinator.deleteGalleryItem`.

## Fix

Added a destructive `Delete` action (`Label("Delete", systemImage: "trash")`) to the gallery row's context menu on both platforms in `GalleryListView`. It sets a new `@State private var itemToDelete: GalleryItem?`, which drives a `confirmationDialog` matching the variant-delete pattern in `GalleryDetailView`: title "Delete this image?", a message naming the item and noting its variants are deleted too and it can't be undone, and a destructive "Delete Image" button that calls `coordinator.deleteGalleryItem(item)`. The iOS swipe `.onDelete` path is left unconfirmed since swipe-to-delete is a deliberate gesture.
