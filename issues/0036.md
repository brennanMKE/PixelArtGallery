# 0036 — Disable App Sandbox for the macOS build (direct distribution)

| | |
|---|---|
| **Status** | open |
| **Module** | Build |
| **Platform** | macOS |
| **First seen** | 2026-07-06 |

## Description

The Mac app will be distributed directly from our website (signed + notarized), not through the App Store, and the decision is to ship it **unsandboxed**. The target currently builds with `ENABLE_APP_SANDBOX = YES` (plus `ENABLE_USER_SELECTED_FILES = readonly`) in the pbxproj target settings. Turn the sandbox off for macOS while keeping the hardened runtime on (notarization requires it) and leaving the iOS build untouched (iOS is always sandboxed by the OS regardless of this setting).

## Expected behavior

- macOS builds produce an unsandboxed app: `codesign -d --entitlements - PixelArtGallery.app` shows no `com.apple.security.app-sandbox` entitlement.
- `ENABLE_HARDENED_RUNTIME = YES` remains in effect for macOS.
- iOS build settings and behavior are unchanged.
- File import/export and network send (FT displays, mDNS) keep working on macOS without sandbox entitlements.

## Notes

- Lesson from #0023: target-level pbxproj settings override xcconfig values. Either edit the pbxproj lines directly or remove them and move the setting into `Configuration/macOS.xcconfig` (`ENABLE_APP_SANDBOX[sdk=macos*] = NO`) — but don't set the xcconfig while the pbxproj still says YES.
- `ENABLE_USER_SELECTED_FILES` is a sandbox entitlement and becomes moot once unsandboxed; remove or leave harmless per what the entitlements dump shows.
- Verify with a macOS build plus an entitlements dump of the built app; run the package tests and an iOS build to confirm no cross-platform regression.
