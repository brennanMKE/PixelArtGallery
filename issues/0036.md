## Description

The Mac app will be distributed directly from our website (signed + notarized), not through the App Store, and the decision is to ship it **unsandboxed**. The target currently builds with `ENABLE_APP_SANDBOX = YES` (plus `ENABLE_USER_SELECTED_FILES = readonly`) in the pbxproj target settings. Turn the sandbox off for macOS while keeping the hardened runtime on (notarization requires it) and leaving the iOS build untouched (iOS is always sandboxed by the OS regardless of this setting).

## Expected behavior

- macOS builds produce an unsandboxed app: `codesign -d --entitlements - PixelArtGallery.app` shows no `com.apple.security.app-sandbox` entitlement.
- `ENABLE_HARDENED_RUNTIME = YES` remains in effect for macOS.
- iOS build settings and behavior are unchanged.
- File import/export and network send (FT displays, mDNS) keep working on macOS without sandbox entitlements.

## Notes

- Lesson from #0023: target-level pbxproj settings override xcconfig values. Either edit the pbxproj lines directly or remove them and move the setting into `Configuration/macOS.xcconfig` (`ENABLE_APP_SANDBOX[sdk=macos*] = NO`) — but don't set the xcconfig while the pbxproj still says YES.
- `ENABLE_USER_SELECTED_FILES` is a sandbox entitlement and becomes moot once unsandboxed; remove or leave harmless per what the entitlements dump shows.
- Verify with a macOS build plus an entitlements dump of the built app; run the package tests and an iOS build to confirm no cross-platform regression.

## Root cause

Not a bug — a distribution decision. The target-level Debug and Release `buildSettings` in `PixelArtGallery.xcodeproj/project.pbxproj` carried `ENABLE_APP_SANDBOX = YES` and `ENABLE_USER_SELECTED_FILES = readonly`, so every macOS build was sandboxed. Since the app ships directly (signed + notarized Developer ID, not App Store), the sandbox is not required and was to be turned off. Per the #0023 lesson, target-level pbxproj settings override xcconfig values, so simply adding an xcconfig line while the pbxproj still said YES would have done nothing.

## Fix

- Removed `ENABLE_APP_SANDBOX = YES;` and `ENABLE_USER_SELECTED_FILES = readonly;` from both the Debug and Release target build configurations in `project.pbxproj`, keeping `ENABLE_HARDENED_RUNTIME = YES;` (notarization requires it).
- Added `ENABLE_APP_SANDBOX[sdk=macos*] = NO` to `Configuration/macOS.xcconfig` so the macOS intent is explicit and future target-level edits are the only way to regress it.
- `ENABLE_USER_SELECTED_FILES` was removed entirely: it only emits the sandbox-scoped `com.apple.security.files.user-selected.*` entitlement, which is moot for an unsandboxed app (confirmed absent from the entitlements dump).
- No `.entitlements` file exists in the project; entitlements are synthesized from build settings, so no other file needed changes. iOS settings were not touched (iOS is always OS-sandboxed regardless).

## Verification

- `xcodebuild -project PixelArtGallery.xcodeproj -scheme PixelArtGallery -destination 'platform=macOS' -showBuildSettings | grep -E 'ENABLE_APP_SANDBOX|ENABLE_HARDENED_RUNTIME'` → `ENABLE_APP_SANDBOX = NO`, `ENABLE_HARDENED_RUNTIME = YES` (no `ENABLE_USER_SELECTED_FILES` line remains).
- `xcodebuild ... -destination 'platform=macOS' build` (automatic signing, "Apple Development: Brennan Stehling") → BUILD SUCCEEDED; codesign invocation shows `-o runtime` (hardened runtime).
- `codesign -d --entitlements - .../Debug/PixelArtGallery.app` → only `com.apple.security.get-task-allow` (Debug signing); **no** `com.apple.security.app-sandbox` and no user-selected-files entitlements.
- `xcodebuild ... -destination 'platform=iOS Simulator,name=iPhone 17 Pro' CODE_SIGNING_ALLOWED=NO build` → BUILD SUCCEEDED.
- `cd PixelArtGalleryKit && swift test` → 92 tests, 0 failures.
- Launched the built macOS app: it starts, the Gallery window renders (empty-state with Import Image), then quit cleanly.

## Files changed

- `PixelArtGallery.xcodeproj/project.pbxproj` — removed `ENABLE_APP_SANDBOX = YES;` and `ENABLE_USER_SELECTED_FILES = readonly;` from the target's Debug and Release `buildSettings`.
- `Configuration/macOS.xcconfig` — added `ENABLE_APP_SANDBOX[sdk=macos*] = NO`.

## Gotchas

- The Debug entitlements dump still shows `com.apple.security.get-task-allow` — that comes from Debug code signing (debugger attach), not the sandbox; a Release/Developer ID build will not have it.
- If Xcode's Signing & Capabilities UI is ever used to re-add a sandbox capability, it will write target-level pbxproj settings that silently beat the xcconfig `NO` (same precedence trap as #0023).
