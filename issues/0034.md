# 0034 — FileStorageManagerTests leak fixture files into the real Application Support directory

| | |
|---|---|
| **Status** | open |
| **Module** | Persistence |
| **Platform** | All |
| **First seen** | 2026-07-06 |

## Description

Running the package test suite pollutes the user's real gallery storage: `FileStorageManagerTests` exercises `FileStorageManager` against its default location, which outside the app sandbox is `~/Library/Application Support/PixelArtGallery/Images`, and tests like `testSaveAndLoadImageData` never delete the files they write. Every `swift test` run leaves small fixture `.dat`/image files behind in the directory the real app reads, and could in principle collide with real user data. Discovered while verifying #0033.

## Expected behavior

Tests run against an isolated temporary directory (e.g. `FileManager.temporaryDirectory` subfolder created per test/suite and removed in teardown), or at minimum delete every file they create. A `swift test` run leaves the user's Application Support untouched.

## Actual behavior

Fixture files accumulate in `~/Library/Application Support/PixelArtGallery/Images` after each test run.

## Notes

- `PixelArtGalleryKit/Tests/PixelArtGalleryKitTests/Persistence/FileStorageManagerTests.swift` — the leaking tests.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/Persistence/FileStorageManager.swift` — check whether it can be initialized with a custom base directory; if not, adding an injectable base URL (defaulting to the current behavior) is the clean fix and keeps production behavior unchanged.
