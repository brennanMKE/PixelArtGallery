## Description

Turn the variant **Preview** (today a read-only inspector — zoom/pan + tap-to-read a pixel's coordinate/RGBA) into a minimal editor: **tap a pixel → pick a color → set it → save**. This is v1 — no drag-paint, no undo/redo, no palette/eyedropper yet (those are follow-ons). It completes the "customize the pixels and save a variant" loop and pairs with the built-in sprites ([#0074](0074.md)): start from a sprite, tweak a few pixels, save your own variant.

## Long Description

The Preview section in `VariantDetailView` renders `PixelGridView` (backed by `PixelGridViewModel`), which decodes the variant's `pixelGridData`, supports zoom/pan, and lets the user tap a pixel to read its coordinate + RGBA. It has no write path. v1 adds: choose a color for the selected pixel and apply it, with an explicit **Save** that persists the edited grid back to the variant. The existing inspect (tap-to-read) behavior must be preserved — use a mode toggle so tapping doesn't accidentally paint.

## Plan

Minimal paint v1: tap a pixel → pick a color → set it → save, plus the two user-requested additions — quick-pick swatches of the variant's most-common colors, and a one-shot **Replace All** color swap. No drag-paint, no undo/redo, no eyedropper, no arbitrary saved palettes, no fill/line — explicit follow-ons. Default mode is Inspect so all current behavior is preserved verbatim.

### Architecture at a glance

`PixelGridViewModel.pixelData: [PixelColor]` (row-major, decoded from `variant.pixelGridData` in `init(variant:)`) **is already the working copy** — `drawPixelGrid` renders from it via `pixelColor(x:y:)`. So the write path is: mutate `pixelData` in place (live preview falls out of `@Observable` for free), track a `hasUnsavedEdits` flag, and only on explicit Save re-encode to RGBA8888 and persist through a new coordinator method. The SwiftData model is never touched until Save; leaving the view discards edits automatically because the view model lives in `PixelGridView`'s `@State`.

### 1. `PixelGridViewModel` — write path, dirty flag, encode, palette + replace helpers

File: `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ViewModels/PixelGridViewModel.swift`

- Add `var hasUnsavedEdits = false`.
- Add the single-pixel paint mutation (mirrors the bounds checks in `pixelColor(x:y:)`):

  ```swift
  /// Set one pixel in the in-memory working copy. No-op (and no dirty flag)
  /// when out of bounds or when the color is unchanged.
  func setPixel(x: Int, y: Int, color: PixelColor) {
      guard x >= 0, x < gridWidth, y >= 0, y < gridHeight else { return }
      let index = y * gridWidth + x
      guard index < pixelData.count, pixelData[index] != color else { return }
      pixelData[index] = color
      hasUnsavedEdits = true
  }
  ```

- Add the encode step, reusing `PixelGrid.toRGBA8888()` (chunk `pixelData` into `gridWidth`-sized rows, build `PixelGrid(width:height:colors:)`, return `toRGBA8888()`):

  ```swift
  /// Encode the working copy back to RGBA8888 bytes for persistence.
  func encodedPixelGridData() -> Data
  ```

- **Quick-pick palette (pure, testable)** — a static helper plus a stored property computed once at init so the swatch row stays stable while the user paints:

  ```swift
  /// Top-`max` distinct colors by frequency, most-frequent first.
  /// Ties break deterministically by first appearance in `pixels`.
  static func dominantColors(in pixels: [PixelColor], max: Int) -> [PixelColor]

  /// Swatches for the quick-pick row, captured from the variant's colors at
  /// load time (deliberately NOT recomputed as the user paints).
  let swatchColors: [PixelColor]   // = Self.dominantColors(in: pixelData, max: 10)
  ```

  Implementation: count with `[PixelColor: Int]` (`PixelColor` is `Hashable`), also record each color's first index; sort by (count desc, firstIndex asc); prefix `max`. Recommend `max: 10`. Both inits populate it (`init(gridWidth:gridHeight:pixelData:)` computes from whatever it's given; empty data → empty swatches).

- **Replace All (pure core + vm mutation)**:

  ```swift
  /// Every exact-RGBA match of `from` becomes `to`; all other pixels untouched.
  static func replacingColor(in pixels: [PixelColor], from: PixelColor, to: PixelColor) -> [PixelColor]

  /// Apply Replace All to the working copy; sets the dirty flag only when
  /// at least one pixel actually changed (from == to or no matches → no-op).
  func replaceAll(of from: PixelColor, with to: PixelColor)
  ```

  Exact RGBA equality is correct here — pixel art uses flat colors.

No other view-model changes — zoom/pan/selection/layout stay as they are.

### 2. `Color` → `PixelColor` bridge

New file: `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/PixelColor+SwiftUI.swift` (UI folder because it imports SwiftUI; `PixelColor` itself stays SwiftUI-free).

- **Testable core** — a quantizing initializer on `PixelColor` taking unit-range components (clamp then round, because `Color.Resolved` components are extended-sRGB and can fall outside 0…1):

  ```swift
  extension PixelColor {
      /// Build from unit-range sRGB components, clamping to 0...1 and
      /// rounding to the nearest 8-bit value.
      init(red01: Double, green01: Double, blue01: Double, opacity01: Double)
  }
  ```

- **Resolved bridge** — `init(resolved: Color.Resolved)` forwarding `Double(resolved.red)` etc. into the above. (iOS 17+/macOS 14+ API; package targets iOS 18/macOS 15, so no availability guards.) The view obtains a `Color.Resolved` via `paintColor.resolve(in: environment)` with `@Environment(\.self) private var environment` — cross-platform, no `UIColor`/`NSColor` needed.
- **Reverse direction** — promote `PixelGridView`'s private `swiftUIColor(_:)` into this extension as `var swiftUIColor: Color` on `PixelColor` (used to render, to seed the picker, and to draw swatches); update `PixelGridView`'s call sites and delete the private helper.

### 3. `PixelGridView` — mode toggle, ColorPicker, swatches, paint tap, Replace All, Save

File: `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/PixelGridView.swift`

- **Init**: `init(variant: Variant? = nil, coordinator: GalleryCoordinator? = nil)`, storing both. Editing UI appears only when *both* are non-nil (`isEditable`). The `#Preview` and any variant-less call sites compile unchanged and stay read-only.
- **View state**: `@State private var isPainting = false` (mode; default Inspect) and `@State private var paintColor: Color = .white`. Mode lives in the view, not the view model — the view model's testable surface stays `Color`-free.
- **Edit controls** (shown only when `isEditable`), placed between `selectionReadout` and the canvas:
  - A segmented `Picker` bound to `isPainting`: **Inspect** / **Paint**. On switching *into* Paint, seed `paintColor` from the currently selected pixel if there is one (`viewModel.pixelColor(...).swiftUIColor`); otherwise keep the previous paint color.
  - `ColorPicker("Paint color", selection: $paintColor, supportsOpacity: true)` (compact label) — visible only in Paint mode.
  - **Swatch row** (Paint mode only): a single `HStack` (wrap in `ScrollView(.horizontal)` for safety) of `viewModel.swatchColors`, each a ~24pt rounded rect styled like the readout swatch (fill `color.swiftUIColor`, hairline `.secondary` stroke), tappable → `paintColor = color.swiftUIColor`. Use `Button` + `.buttonStyle(.plain)` so it works on both platforms.
  - **Replace All** button (Paint mode only): `.disabled(viewModel.selectedPixel == nil)`. Action: `from = viewModel.pixelColor(x: sel.x, y: sel.y)` (the selected pixel's *current* color, shown in the readout), `to = PixelColor(resolved: paintColor.resolve(in: environment))`, then `viewModel.replaceAll(of: from, with: to)`. Live preview + dirty flag come from the vm.
  - A **Save** button, `.disabled(!viewModel.hasUnsavedEdits)`, plus a subtle unsaved indicator (small filled circle or "Edited" caption while `hasUnsavedEdits`).
- **Tap handling**: in `tapGesture(container:)`'s `onEnded`, branch on mode:

  ```swift
  if isPainting {
      if let coord = viewModel.gridCoordinate(at: value.location, in: container) {
          viewModel.setPixel(x: coord.x, y: coord.y, color: PixelColor(resolved: paintColor.resolve(in: environment)))
          viewModel.selectPixel(x: coord.x, y: coord.y)   // readout shows the new value
      }                                                    // outside the grid: no-op
  } else {
      viewModel.selectPixel(at: value.location, in: container)  // current behavior
  }
  ```

- **Zoom/pan coexistence**: no gesture changes needed. `SpatialTapGesture` (discrete) vs `DragGesture(minimumDistance: 10)` already disambiguate — a drag ≥10pt pans and never fires the tap; a clean tap paints. Pinch/buttons/Size-to-Fit untouched, so painting at 40× zoom works today's way.
- **Save action**: `try coordinator.updateVariantPixels(variant, pixelGridData: viewModel.encodedPixelGridData())`, then `viewModel.hasUnsavedEdits = false`. Errors surface via the coordinator's existing `currentError` pipeline; no bespoke alert in v1.
- **Discard-on-leave (decision): plain discard, no prompt.** The working copy is `@State`; navigating away tears it down and the variant is untouched. The unsaved indicator is the safeguard. A confirmation dialog is deliberate non-scope for v1.

### 5. `GalleryCoordinator.updateVariantPixels`

File: `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ViewModels/GalleryCoordinator.swift`

Modeled directly on `updateVariantDimensions(_:width:height:)` minus the engine work (no I/O → synchronous, not async):

```swift
/// Persist an edited pixel grid onto an existing variant (#0076).
func updateVariantPixels(_ variant: Variant, pixelGridData: Data) throws {
    guard let modelContext else { … throw GalleryCoordinatorError.missingModelContext }
    let expected = variant.targetWidth * variant.targetHeight * 4
    guard pixelGridData.count == expected else {
        throw PixelGridError.invalidDataSize(expected: expected, actual: pixelGridData.count)   // defensive; reuse existing error
    }
    variant.pixelGridData = pixelGridData
    do { try modelContext.save(); AppLog.variant.info(…) }
    catch { currentError = error.localizedDescription; AppLog.variant.error(…); throw error }
}
```

Variants are the user's — no `isBuiltIn` guard here (editing a built-in *sprite's variant* is expressly fine; only built-in gallery items themselves are protected, and they already are).

## Root cause

Not a bug — a feature gap. `PixelGridView`/`PixelGridViewModel` decoded a
variant's `pixelGridData` into a working `[PixelColor]` array purely for
rendering and read-only inspection (zoom/pan/tap-to-read coordinate+RGBA).
There was no write path: no way to mutate a pixel, no dirty tracking, no way
to re-encode the working copy back to `pixelGridData`, and no coordinator
method to persist an edited grid. `VariantDetailView`'s Preview section
instantiated `PixelGridView(variant:)` with no coordinator, so even a future
write path in the view model would have had nowhere to save through.

## Fix

Implemented per the `## Plan`, no material divergences:

- **`PixelGridViewModel`** (`ViewModels/PixelGridViewModel.swift`): added
  `hasUnsavedEdits`, `setPixel(x:y:color:)` (bounds-checked, no-op when
  out-of-bounds or color unchanged), `encodedPixelGridData()` (chunks
  `pixelData` into `gridWidth`-wide rows, builds a `PixelGrid`, re-encodes via
  `toRGBA8888()`), `static dominantColors(in:max:)` (frequency count + first-

## Gotchas

- `PixelGridViewModel.init(variant:)` had to compute the flattened
  `pixelData` into a local `let` before assigning `self.pixelData` and
  `self.swatchColors` — assigning `self.swatchColors` directly from
  `Self.dominantColors(in: self.pixelData, ...)` failed to compile
  ("'self' used before all stored properties are initialized") because
  `swatchColors` hadn't been assigned yet at that point in the initializer.
- `updateVariantPixels` is synchronous (unlike the `async`
  `updateVariantDimensions`) since there's no `PixelationEngine`/file I/O
  involved — just a byte-count check and a `modelContext.save()` — matching
  the plan's "minus the engine work" note.
