## Description

The Mac app ships via direct download, so it needs a public site serving three purposes: a marketing/landing page with screenshots and a download link, the Sparkle appcast endpoint (`appcast.xml`) the app polls for updates, and the DMG downloads themselves. Static HTML/CSS in a new `website/` folder at the repo root — no framework, no build step — plus a deploy script.

## Acceptance criteria

- `website/` contains:
  - `index.html` — what Pixel Art Gallery is (import → pixelate → send to Flaschen Taschen displays), screenshots, a prominent Download button pointing at the latest DMG, system requirements (macOS 15+), link to changelog and privacy pages.
  - `changelog.html` — versioned release notes (one `<article id="vX-Y-Z">` per release), linked from the appcast items.
  - `privacy.html` — privacy policy (the app talks to local-network FT displays and optionally saves to Photos on iOS; no analytics/tracking).
  - `appcast.xml` — Sparkle feed, newest item first; items are generated by #0041, not hand-typed.
  - `downloads/` — versioned DMGs (`PixelArtGallery-<X.Y.Z>.dmg`).
  - `assets/` — app icon renders (512/1024), OpenGraph card, and real app screenshots (the grid gallery, a variant with pixel preview, send-to-display).
- `scripts/deploy-website.sh` pushes `website/` to the host via rsync over SSH, driven by env vars (e.g. `PAG_EC2_KEY`, `PAG_EC2_HOST`, `PAG_EC2_PATH`, optional `PAG_EC2_PORT`), validating env vars and key permissions before pushing.
- Site works as plain static files (opens correctly from `file://` or any static host), light/dark friendly, presentable on mobile.
