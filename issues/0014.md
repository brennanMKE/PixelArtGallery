## Description

The PRD lists duplicate prevention ("Option to skip importing images already in gallery") as a gallery storage feature. Import currently always creates a new `GalleryItem` with no de-duplication check.

## Expected behavior

On import, the app detects whether an identical image already exists (e.g. by content hash of the original bytes or dimensions+name) and offers to skip rather than creating a duplicate gallery item.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ViewModels/GalleryCoordinator.swift` — `createGalleryItem`; add a hash/identity check before insert.
- Consider storing a content hash on `GalleryItem` to make the check cheap.
- PRD §1 "Duplicate Prevention". Depends on 0003 (real import path).

## Root cause

`GalleryCoordinator.createGalleryItem(name:imageData:)` (landed in 0003) always
wrote the bytes to disk and inserted a new `GalleryItem`, with no identity check.
Re-importing the same image therefore produced duplicate items and duplicate
on-disk files. `GalleryItem` had no stored identity for the original bytes to
make such a check cheap.

## Fix

Added a `contentHash` property to `GalleryItem` (lowercase hex SHA-256 of the
original bytes) with a safe `""` default so records persisted before the field
existed still load. `createGalleryItem` now hashes the incoming bytes first
(via CryptoKit) and runs a `FetchDescriptor` `#Predicate` for an existing item
with the same hash; on a match it skips entirely — no file written, no insert —
returns the new `ImportResult.duplicate(existingName:)`, and sets a non-error
`importMessage`. A fresh image writes the file, records the hash, inserts, and
returns `.created`. `GalleryListView` surfaces `importMessage` via an
informational "Import" alert (separate from the error alert). Manual display
entry and variant creation are untouched.
