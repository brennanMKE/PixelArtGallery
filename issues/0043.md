## Description

The release pipeline has many independent gates (clean build, green tests, version bumped, Sparkle wired, signing cert present, notary profile present, deploy env vars set, clean working tree). Checking them by hand before `release.sh` invites skipped steps. A single `scripts/preflight.sh` runs every gate in one pass and fails loudly on the first `[✗]`, turning "is this shippable right now?" into a 30-second command.

## Acceptance criteria

`scripts/preflight.sh` reports pass/fail per gate:

- **Build gates** — macOS build succeeds; `swift test` in `PixelArtGalleryKit` passes (with an observed test count, not just exit code); iOS build succeeds (multiplatform target must not regress).
- **Version gates** — `MARKETING_VERSION` present only in `Configuration/Build.xcconfig` (any `MARKETING_VERSION` line in `project.pbxproj` is a failure, per #0042); value matches SemVer; value is strictly greater than the newest version in `website/appcast.xml`.
- **Sparkle gates** — Sparkle dependency present; built macOS bundle's Info.plist has non-empty `SUFeedURL` and `SUPublicEDKey` (`plutil -extract … raw`); the iOS build does NOT embed Sparkle.
- **Sandbox gate** — built macOS app's entitlements show no `com.apple.security.app-sandbox` (#0036) and hardened runtime is on.
- **Signing gates** — a `Developer ID Application` certificate is in the keychain; the `PixelArtGallery-notary` notarytool profile exists.
- **Website gates** — deploy env vars set and the SSH key file has `600` permissions (#0040); newest appcast item's DMG exists in `website/downloads/` and its advertised version/length match the real artifact (mount and compare — the appcast↔DMG consistency check).
- **Hygiene gates** — clean working tree (`--allow-dirty` to bypass for dry runs); `--skip-build` and `--strict` flags supported.

## Notes

- Depends on #0036–#0042; this lands last among the scripts and encodes their invariants.
- Keep each gate's failure message actionable (what's wrong + the command that fixes it).

## Root cause

Feature gap: the release pipeline's gates (#0036–#0042) each lived in their own script or convention with no single command that verified all of them before `scripts/release.sh`. Checking by hand invited skipped steps.

## Fix

Added `scripts/preflight.sh`, which runs every gate with `[✓]`/`[✗]`/`[!]`/`[-]` output, an actionable fix hint on each failure, a summary count, and a nonzero exit if any gate fails:

- **Build** — macOS build; `swift test` in `PixelArtGalleryKit` with the executed-test count parsed from output (sums the XCTest "Executed N tests" total and Swift Testing "Test run with N tests" count, requires > 0); iOS Simulator build. Builds use the fixed derived-data path `build/preflight` so built-product gates inspect exactly what was just built.
- **Version** — `MARKETING_VERSION` present in `Configuration/Build.xcconfig` and absent from `project.pbxproj` (#0042); SemVer `X.Y.Z`; strictly greater than the newest `sparkle:shortVersionString` in `website/appcast.xml` (zero items = pass with a first-release note).
- **Sparkle** — dependency in `PixelArtGalleryKit/Package.swift`; built macOS app has non-empty `SUFeedURL` and `SUPublicEDKey` via `plutil -extract … raw`; built iOS app has no `*sparkle*` files (#0038/#0039).
- **Sandbox** — built macOS app's entitlements have no `com.apple.security.app-sandbox` (#0036); `ENABLE_HARDENED_RUNTIME = YES` in `-showBuildSettings`.
- **Signing** — `security find-identity` lists a `Developer ID Application` cert; `xcrun notarytool history --keychain-profile PixelArtGallery-notary` works. A missing profile (keychain error) is a failure pointing at `scripts/setup-keys.sh`; other errors (likely network) are warnings.
- **Website** — `PAG_EC2_HOST`/`PATH`/`KEY` set, key file exists with 600/400 perms (#0040); newest appcast item's DMG exists in `website/downloads/` with matching enclosure length, and the DMG is mounted read-only to compare the real `CFBundleShortVersionString` against the advertised one (#0041). Zero items = pass with note.
- **Hygiene** — `git status --porcelain` empty; `--allow-dirty` downgrades to a warning.

Flags: `--skip-build` (build gates report `[-] skipped`, built-product gates probe the last `build/preflight` products if present), `--strict` (warnings become failures), `--allow-dirty`.

## Gotchas

- `swift test` runs XCTest and Swift Testing in one invocation; the Swift Testing harness prints "Test run with 0 tests … passed" even when all 92 tests are XCTest, so the count parser sums the largest XCTest "Executed N tests, with 0 failures" line with the Swift Testing count instead of trusting either line alone.
- The notary gate distinguishes a missing keychain profile (fail, actionable) from other notarytool errors (warn — likely network), so an offline run doesn't hard-fail on credentials that actually exist.
- Built-product gates (Sparkle plist keys, iOS Sparkle absence, entitlements) depend on the fixed `-derivedDataPath build/preflight`; under `--skip-build` they probe the previous preflight build and report `[-] skipped` when none exists.
