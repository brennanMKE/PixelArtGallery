# 0052 — Continuous send keeps running after leaving the variant view; it should stop on disappear

| | |
|---|---|
| **Status** | open |
| **Module** | Networking / UI |
| **Platform** | All |
| **First seen** | 2026-07-07 |

## Description

When the continuous send is active (the Send button shows "Stop Sending") and the user navigates away from the variant detail view without tapping Stop, the send loop keeps running in the background — it should stop automatically when the view is left, on both macOS and iPhone.

## Steps to reproduce

1. Open a variant's detail view and pick a display.
2. Tap **Send Now** so it enters the continuous "Stop Sending" state.
3. Navigate back / dismiss the view (iPhone: back button or swipe; macOS: pop the navigation stack or close the window) without tapping Stop.
4. The refresh loop keeps sending frames to the display in the background.

## Expected behavior

Leaving the view cancels the in-flight continuous send, so no further frames are pushed once the view is gone — on both macOS and iPhone.

## Actual behavior

The send `Task` is retained in `@State var sendTask` (`UI/VariantDetailView.swift`) and is only cancelled by tapping Stop (`sendTask?.cancel()`). There is no `onDisappear`/lifecycle hook, so the loop outlives the view and keeps sending.

## Notes

- Add an `.onDisappear { sendTask?.cancel() }` on `VariantDetailView` (and set `isSending = false`), or drive the loop with SwiftUI's `.task { }` modifier, which SwiftUI cancels automatically when the view disappears — likely the cleaner fix, though it changes how the loop is started (bound to a state flag rather than a button-launched `Task`).
- Verify on both platforms: `.onDisappear` fires on iOS navigation pop and on macOS when the view leaves the hierarchy. Watch for the macOS case where the detail view can be replaced in a `NavigationSplitView` detail column.
- Cancellation is already cooperative (#0050/#0051): cancelling the task ends the loop and aborts any in-flight packet, so this is purely about triggering that cancel on view exit.

## Relation

- Pairs with: [#0050](0050.md) — introduced the retained continuous-send `Task`; this issue bounds its lifetime to the view.
