# 0002 — Connect GalleryCoordinator to the SwiftData ModelContext for live queries and persistence

| | |
|---|---|
| **Status** | open |
| **Module** | ViewModels / Persistence |
| **Platform** | All |
| **First seen** | 2026-06-25 |

## Description

`GalleryCoordinator` holds gallery items in a plain in-memory array (`var galleryItems: [GalleryItem] = []`) that nothing ever populates, and its create/delete methods don't touch a `ModelContext`. As a result nothing persists and the list is always empty. The coordinator needs access to the SwiftData context so items can be fetched, inserted, and deleted for real.

## Expected behavior

Gallery items are read from SwiftData (via `@Query` in the view feeding the coordinator, or a `ModelContext` injected into the coordinator) and reflect inserts/deletes immediately. `deleteGalleryItem`/`deleteVariant` remove objects from the context. Data survives app relaunch.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ViewModels/GalleryCoordinator.swift` — the previous `@Query` property (invalid in an `@Observable` class) was replaced with a stub array during the build fix; this issue makes the data flow real.
- `@Query` belongs in a SwiftUI `View`; decide whether the view owns the query and assigns into the coordinator, or the coordinator fetches from an injected context. Keep `@Observable` (never `ObservableObject`) per PRD.
- Prerequisite for 0003, 0004, 0014, 0015.
