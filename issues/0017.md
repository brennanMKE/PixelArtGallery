# 0017 — Imported image never displays; gallery thumbnail and detail preview show a placeholder icon

| | |
|---|---|
| **Status** | open |
| **Module** | UI |
| **Platform** | All |
| **First seen** | 2026-06-25 |

## Description

After importing an image, it does not show at all on macOS or iOS — the gallery row and the item detail screen display a generic `photo.fill` SF Symbol instead of the imported picture. Import itself works (bytes are saved and dimensions are read), so the original image is on disk but never loaded into the UI.

## Steps to reproduce

1. Launch the app (Mac or iPhone Simulator).
2. Import an image.
3. Observe the gallery list row and open the item's detail view.

## Expected behavior

The gallery row shows a thumbnail of the imported image, and the detail view's "Original Image" section shows the actual image.

## Actual behavior

Both show a static `Image(systemName: "photo.fill")` placeholder. The real image is never rendered.

## Notes

- Import is confirmed working from the logs: `Created gallery item: Imported Image (4288×2848) at 1DE8D4CE-…​.dat` — bytes saved via `FileStorageManager`, dimensions correct.
- Root cause: the views never load the stored file. `GalleryListView.swift` (thumbnail, ~line 57) and `GalleryDetailView.swift` (original preview, ~line 22) both hardcode `Image(systemName: "photo.fill")` and ignore `item.originalImagePath`.
- Fix needs to load bytes from `FileStorageManager` (actor; `load(filename:)`) keyed on `originalImagePath` and render a cross-platform image (`NSImage`/`UIImage`). Thumbnails should downsample the (potentially 12MP) original rather than decoding it at full size for a 60×60 cell.
