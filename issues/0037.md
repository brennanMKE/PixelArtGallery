## Description

Set up code signing with a Developer ID Application certificate and Apple's notary service so direct downloads of the Mac app aren't blocked by Gatekeeper, wrapped in a scripted pipeline so a release is a single command instead of an error-prone manual sequence (archive → export → sign → notarize → wait → staple → DMG → verify).

## Acceptance criteria

- `scripts/release.sh` produces a signed, notarized, stapled DMG from a clean checkout: archives the `PixelArtGallery` scheme for `platform=macOS` (Release), exports with Developer ID signing, submits via `xcrun notarytool submit --wait`, staples, packages a DMG at `dist/PixelArtGallery-<version-or-sha>.dmg`, and verifies.
- `scripts/setup-keys.sh` does the one-time `xcrun notarytool store-credentials` setup for a `PixelArtGallery-notary` keychain profile (app-specific Apple ID password; never committed).
- `scripts/verify-dmg.sh <path>` independently confirms signing, notarization, stapling, and Gatekeeper acceptance under quarantine (`spctl --assess`).
- The scripts fail early with actionable errors when the Developer ID certificate or notary profile is missing.
- `dist/` is gitignored.
- The signed app passes `spctl --assess --verbose=4` and opens on a clean Mac with no "unidentified developer" warning.

## Root cause

Feature gap, not a bug: the project had no release tooling at all — producing a Gatekeeper-clean direct download required a long manual sequence (archive with the right destination, Developer ID export, notarize, wait, staple, DMG, sign, verify), each step easy to get wrong or skip.

## Fix

New `scripts/` directory with the full pipeline:

- `scripts/release.sh` — preflight (Developer ID Application cert via `security find-identity`; `PixelArtGallery-notary` keychain profile via a `notarytool history` probe, skipped in dry-run) → `xcodebuild archive` (`PixelArtGallery` scheme, Release, `-destination 'generic/platform=macOS'`, `CURRENT_PROJECT_VERSION=$(date -u +%Y%m%d)` override) → `xcodebuild -exportArchive` with `scripts/ExportOptions.plist` (method `developer-id`, team `XV8BAAVZ6V`) → `codesign --verify --deep --strict` + Developer ID authority check → `notarytool submit --wait` (app zip) → `stapler staple` app → `hdiutil` DMG at `dist/PixelArtGallery-<MARKETING_VERSION>.dmg` with app + `/Applications` symlink → sign DMG → `notarytool submit --wait` (DMG, needed so the DMG itself can be stapled) → `stapler staple` DMG → `scripts/verify-dmg.sh` → prints build number and next steps. `MARKETING_VERSION` is read from `xcodebuild -showBuildSettings` (never hard-coded, ready for #0042). `--dry-run`/`--skip-notarize` stops after export + codesign verification and never contacts Apple. xcodebuild output goes to log files; on failure the log tail is surfaced.
- `scripts/setup-keys.sh` — interactive one-time `xcrun notarytool store-credentials PixelArtGallery-notary --team-id XV8BAAVZ6V` with app-specific-password guidance, then verifies the profile with a `notarytool history` probe. Secrets live only in the keychain.
- `scripts/verify-dmg.sh <path>` — six checks with per-check PASS/FAIL and nonzero exit on any failure: `spctl --assess --type open --context context:primary-signature` on the DMG, `stapler validate` on the DMG, mounts the DMG (`hdiutil attach`, detach on trap), `codesign --verify --deep --strict` on the app, Developer ID authority check, `stapler validate` on the app, and `spctl --assess --type execute` on a copy carrying a freshly written `com.apple.quarantine` xattr.
- `.gitignore` gains `dist/` and `*.xcarchive`.

## Gotchas

- With `set -o pipefail`, `codesign ... | grep -q` fails spuriously: `grep -q` exits on first match, codesign gets SIGPIPE, and the pipeline reports failure. Capture output into a variable first, then grep it.
- A DMG cannot be stapled unless the DMG itself is notarized — stapling the app inside is not enough — so the pipeline submits the DMG in a second (fast) notarization pass before `stapler staple <dmg>`.
- The acceptance criterion "opens on a clean Mac with no unidentified-developer warning" can only be confirmed after a real (user-run) notarized release; the local `spctl` checks are the strongest proxy available without contacting Apple.
- The target is multiplatform: the archive step must keep `-destination 'generic/platform=macOS'` or xcodebuild may pick an iOS destination.
