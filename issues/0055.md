# 0055 — Variant view does not auto-select the display matching the variant's dimensions

| | |
|---|---|
| **Status** | open |
| **Module** | UI |
| **Platform** | All |
| **First seen** | 2026-07-09 |

## Description

When selecting a variant such as 8x8 while a display which is 8x8 exists, the app should automatically select that display in the send UI. Today the user has to change the display selection manually, often, which is cumbersome.

## Steps to reproduce

1. Register two displays with different geometries (e.g. 8x8 and 64x64).
2. Open an 8x8 variant.
3. Look at the display picker in the send section.

## Expected behavior

The display whose geometry matches the variant's dimensions (8x8) is selected automatically when the variant view opens.

## Actual behavior

The previously selected (or default) display stays selected regardless of the variant's dimensions, and the user must switch it manually every time.

## Notes

Likely code: `VariantDetailView.swift` (display picker / send section) and `FlaschenTaschenDisplay` (width/height). If several displays match the variant's dimensions, keep the current selection when it already matches, otherwise pick the first match; fall back to existing behavior when nothing matches.
