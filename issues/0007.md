## Description

Export is entirely faked. `ExportPickerView` and `VariantDetailView` simulate exporting with `DispatchQueue.main.asyncAfter` delays and never write a file. There is no `VariantExporter` type at all — the `Export` module referenced in `PixelArtGalleryKit.swift` was never created. None of the four PRD formats (PNG, HEIC, PPM, JSON color matrix) are actually produced.

## Expected behavior

A real `VariantExporter` converts a `Variant`'s `pixelGridData` into each format and writes it to the chosen URL:
- **PNG** / **HEIC** — encoded raster images (honoring `scaleFactor`).
- **PPM** — raw portable pixmap.
- **JSON** — color-matrix representation.

`ExportPickerView`'s export action calls the exporter instead of simulating, and reports real success/failure.

## Notes

- Create `PixelArtGalleryKit/Sources/PixelArtGalleryKit/Export/VariantExporter.swift` (+ `ExportError`); the `PixelArtGalleryKit.swift` placeholder already documents these as expected public exports.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/ExportPickerView.swift` — replace the simulated `performExport` delay; macOS `NSSavePanel` path already exists, iOS path is a `// TODO` (see 0008).
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/VariantDetailView.swift` — `handleExport` is also simulated.
- PRD §4 "Export". The `MockVariant` used by `ExportPickerView` should be replaced with the real `Variant`.

## Root cause

The `Export` module was never created. Both `ExportPickerView.performExport` and
`VariantDetailView.handleExport` only ran `DispatchQueue.main.asyncAfter` delays and reported a
fabricated success without writing any bytes. `ExportPickerView` also operated on a throwaway
`MockVariant` carrying only dimensions, so it never had access to a variant's `pixelGridData`.

## Fix

Added a public, `nonisolated` `VariantExporter` (+ `ExportError`, `ExportFormat`) under a new
`Export/` directory. It reconstructs a `PixelGrid` from the variant's RGBA8888 `pixelGridData`
via `PixelGrid.fromRGBA8888` and encodes each format: PNG/HEIC through CoreGraphics/ImageIO
(`CGImageDestination`), nearest-neighbor upscaling each source pixel into a
`scaleFactor × scaleFactor` block and premultiplying alpha so the CGContext format is supported;
PPM as a raw `P6` portable pixmap; JSON as a row-major `[r,g,b,a]` color matrix. A value-typed
entry point (`width/height/pixelGridData/scaleFactor`) lets the UI encode inside a detached task
without passing the `@Model` `Variant` (main-actor bound) across actor boundaries. `ExportPickerView`
now takes the real `Variant` and calls the exporter, reporting genuine success/failure;
`VariantDetailView.handleExport` no longer simulates. The iOS save-destination (Photos) TODO is
left untouched for issue 0008; the macOS `NSSavePanel → exporter` path is real.

## Gotchas

- A straight-alpha `CGImageAlphaInfo.last` RGB context is unsupported and makes `makeImage()`
  return nil (`imageCreationFailed`). Use `premultipliedLast` and premultiply channels manually,
  matching the existing `PixelationEngine` pipeline.
- `Variant` is a SwiftData `@Model` and inherits `MainActor` default isolation from the package's
  `defaultIsolation(MainActor.self)` setting, so it cannot be passed into a `Task.detached`. The
  exporter exposes a value-typed overload for that reason.
