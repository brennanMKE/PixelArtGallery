# 0007 — Implement real variant export for PNG, HEIC, PPM, and JSON

| | |
|---|---|
| **Status** | open |
| **Module** | Export |
| **Platform** | All |
| **First seen** | 2026-06-25 |

## Description

Export is entirely faked. `ExportPickerView` and `VariantDetailView` simulate exporting with `DispatchQueue.main.asyncAfter` delays and never write a file. There is no `VariantExporter` type at all — the `Export` module referenced in `PixelArtGalleryKit.swift` was never created. None of the four PRD formats (PNG, HEIC, PPM, JSON color matrix) are actually produced.

## Expected behavior

A real `VariantExporter` converts a `Variant`'s `pixelGridData` into each format and writes it to the chosen URL:
- **PNG** / **HEIC** — encoded raster images (honoring `scaleFactor`).
- **PPM** — raw portable pixmap.
- **JSON** — color-matrix representation.

`ExportPickerView`'s export action calls the exporter instead of simulating, and reports real success/failure.

## Notes

- Create `PixelArtGalleryKit/Sources/PixelArtGalleryKit/Export/VariantExporter.swift` (+ `ExportError`); the `PixelArtGalleryKit.swift` placeholder already documents these as expected public exports.

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/VariantDetailView.swift` — `handleExport` is also simulated.
- PRD §4 "Export". The `MockVariant` used by `ExportPickerView` should be replaced with the real `Variant`.
