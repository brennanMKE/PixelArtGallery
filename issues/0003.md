## Description

Importing an image picks data but never saves it. `GalleryCoordinator.createGalleryItem(name:imageData:)` builds a `GalleryItem` but doesn't write the bytes through `FileStorageManager` and doesn't insert the item into the `ModelContext` — both are `// TODO`s. The picked image is silently dropped.

## Expected behavior

Selecting an image writes the original bytes to disk via `FileStorageManager`, stores the returned filename in `GalleryItem.originalImagePath`, captures real pixel dimensions, inserts the item into SwiftData, and surfaces it in the gallery list. Errors are reported via the existing `currentError` channel.

## Notes

- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/ViewModels/GalleryCoordinator.swift` — `createGalleryItem` (TODO at "Use FileStorageManager to save image data") and `loadImage(from:)` for dimensions.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/Persistence/FileStorageManager.swift` — `save(imageData:)` / `load(filename:)`.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/ImagePickerView.swift` — supplies JPEG/file data on both platforms.
- Depends on 0002 (context wiring).

## Root cause

`createGalleryItem` carried two unfinished TODOs: it never wrote the picked
bytes through `FileStorageManager` and seeded `originalImagePath` with a
throwaway `"<uuid>.jpg"` string that pointed at no real file. The image data
was decoded only long enough to read dimensions and was then discarded, so
nothing about the import was persisted to disk.

## Fix

`createGalleryItem(name:imageData:)` is now `async` (required because
`FileStorageManager` is an actor). It lazily creates and caches a
`FileStorageManager`, saves the original bytes via `save(imageData:)`, and
stores the returned real filename in `GalleryItem.originalImagePath`. Real
pixel dimensions are still read through the existing `loadImage(from:)`
CGImage path, and the item is inserted and saved (0002's path). Disk, decode,
and SwiftData failures are surfaced on `currentError` and rethrown. The
`GalleryListView` import call site (already inside a `Task`) was updated to
`await` the method. Scope was kept to import persistence — no duplicate
detection (0014) or variant generation (0004).
