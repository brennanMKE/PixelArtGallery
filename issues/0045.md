# 0045 — Separate bundle ID and identity for development vs release builds

| | |
|---|---|
| **Status** | open |
| **Module** | Build |
| **Platform** | macOS |
| **First seen** | 2026-07-06 |

## Description

Development builds from Xcode currently use the same bundle ID (`co.sstools.PixelArtGallery`) as the future notarized release, so a dev build and the shipped app share UserDefaults, the SwiftData store/Application Support directory, Sparkle state, and dock identity on the same machine. Give development builds a distinct identity so both coexist cleanly and dev data never contaminates the released app's state.

## Expected behavior

- Xcode/dev builds: bundle ID `co.sstools.PixelArtGallery.beta`, display name "Pixel Art Gallery Beta", a visually distinct app icon variant, and no production Sparkle feed (empty/absent `SUFeedURL` keeps Check for Updates disabled per #0038's `isConfigured` guard).
- Release builds (via `scripts/release.sh` / the release configuration): bundle ID `co.sstools.PixelArtGallery`, name "Pixel Art Gallery", production icon and feed.
- Switching requires no manual file edits — the build configuration/scheme drives it (xcconfig layering; e.g. an uncommitted `Active.xcconfig` including either a Beta or Prod xcconfig, or per-configuration settings).
- Both variants install side-by-side with separate Application Support/SwiftData stores.

## Notes

- The app's data directory is derived from the bundle's Application Support path in `PixelArtGalleryApp.getGalleryDatabaseURL()` — confirm the beta bundle ID actually yields a separate container path when unsandboxed (it uses a shared `~/Library/Application Support/PixelArtGallery` folder name today; the beta variant may need its own folder name driven by bundle ID or an Info.plist value).
- Beta icon: a badged variant of the existing macOS icon set (e.g. `AppIcon-macOS-Beta`) so dev and prod are distinguishable in the Dock.
- Optional but recommended before the first public release; the preflight (#0043) should assert release builds carry the production bundle ID.
- Watch the #0023 precedence rule when moving `PRODUCT_BUNDLE_IDENTIFIER` out of the pbxproj into xcconfigs, and keep the iOS build working (its bundle ID handling can stay as-is or follow the same split).
