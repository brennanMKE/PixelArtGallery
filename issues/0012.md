# 0012 — Send variants to Flaschen Taschen displays over the network

| | |
|---|---|
| **Status** | open |
| **Module** | Networking |
| **Platform** | All |
| **First seen** | 2026-06-25 |

## Description

Sending a variant to a display is faked. `VariantDetailView.handleSendToDisplay()` just waits 1.5s with `DispatchQueue.main.asyncAfter` and shows a fake success message. There is no FT client — the `FTDisplayClient`/`FTDisplayError` types named in `PixelArtGalleryKit.swift` don't exist, and the `ft-swift` library the PRD references is not a package dependency.

## Expected behavior

A real `FTDisplayClient` sends a variant's pixel data to a stored display's host/port (the FT/UDP image protocol, e.g. via `ft-swift`), reporting genuine progress and success/error. The "Send Now" button drives the real client against the selected persisted display.

## Notes

- Create `PixelArtGalleryKit/Sources/PixelArtGalleryKit/Networking/FTDisplayClient.swift` (+ `FTDisplayError`).
- Add `ft-swift` as a dependency in `PixelArtGalleryKit/Package.swift` (currently no external dependencies), or implement the FT protocol directly.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/VariantDetailView.swift` — replace simulated `handleSendToDisplay`.
- PRD §3 "Send to FT" and §"Export & Send". Depends on 0011 for real display selection. Log under `FTDiscovery`/a send category.
