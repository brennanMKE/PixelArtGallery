# 0031 — macOS window has no minimum size and can shrink to an unusable layout

| | |
|---|---|
| **Status** | open |
| **Module** | App / UI |
| **Platform** | macOS |
| **First seen** | 2026-07-06 |

## Description

The `WindowGroup` presents `ContentView` with no minimum frame, so the macOS window can be resized down to a sliver where the gallery list, detail views, and toolbar collapse into an unusable layout. Standard Mac practice is to set a sensible minimum content size.

## Steps to reproduce

1. Run the macOS app.
2. Drag the window's corner to shrink it — it collapses far below a usable size.

## Expected behavior

The window resists shrinking below a reasonable minimum (roughly 640×480 or similar) so all primary screens stay usable.

## Actual behavior

No minimum; content squashes arbitrarily.

## Notes

- Apply `#if os(macOS) .frame(minWidth:…, minHeight:…)` on `ContentView` in `PixelArtGallery/PixelArtGalleryApp.swift` (or on `GalleryListView`). Keep it out of the iOS build path.
