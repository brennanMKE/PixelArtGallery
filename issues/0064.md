## Description

Final phase of the display-first flow ([#0061](0061.md)): the user-facing UI. From a gallery item, present the known FT displays; tapping one auto-creates the aspect-fit, centered variant (via [#0063](0063.md)) and navigates to the send scene with the x/y offset seeded to the centered value, so the source appears pixelated to that display's dimensions ready to send — no dimension entry. The manual width/height form is demoted to a secondary "Custom size…" path (kept, not removed). Manual offset nudges during a send are clamped so the image can't be pushed off the display. Shared views for iOS and macOS with `#if os` tweaks.

## Long Description

Current navigation: `GalleryListView` (grid) → `GalleryDetailView` (item + variant list) → `VariantDetailView` (send, with the #0056/#0057 send section at top and live layer/x/y). Displays are only reachable via Settings → `DisplayRegistryView`, whose rows currently open an *edit* sheet on tap.

New flow entry: from `GalleryDetailView`, a primary "Send to Display" action lists the FT displays (reuse the `@Query`/`DisplayRow` pattern from `DisplayRegistryView`). Tapping a display calls `coordinator.createFittedVariant(for:display:)` ([#0063](0063.md)) and pushes `VariantDetailView` for the resulting variant, with the send controls seeded to the centered offset (via the #0063 helper) and the display preselected. The user can send immediately, adjust live (layer/x/y), edit the pixelation, or keep it — it's a normal saved variant tagged with the display.

## Notes

Relevant code:
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/GalleryDetailView.swift` — `startVariantCreation()` and the "Create Variant" button (the manual entry point to demote); it presents `VariantCreationView` via `coordinator.showVariantCreation`.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/VariantCreationView.swift` — the manual W/H form (keep as secondary "Custom size…").
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/DisplayRegistryView.swift` — `@Query(sort: \FlaschenTaschenDisplay.discoveredDate)` + `DisplayRow` to reuse for the picker list. Rows currently tap to edit.
- `PixelArtGalleryKit/Sources/PixelArtGalleryKit/UI/VariantDetailView.swift` — the send scene. `seedSendDefaults()` (~lines 311–316) currently seeds layer/offsets from the display's stored defaults; this flow must instead seed the **centered** offset from #0063's helper when arriving via display-first. `updateSelectedDisplay()`/`preferredSelection(...)` preselect the display. Offset steppers (`offsetXRange`/`offsetYRange`, ~lines 282–288) currently clamp only to `0...(displayDim - 1)` and ignore image size — add an upper bound so `offset + imageDim ≤ displayDim`.
- Navigation is a single-column `NavigationStack` with value-based `navigationDestination`; all views shared across platforms with inline `#if os(...)`.

Design points (planner to finalize):
- Where to surface the display list: a pushed destination vs a sheet from `GalleryDetailView`. Recommend a pushed `navigationDestination` so tapping a display can push onward to `VariantDetailView` in the same stack. Planner to choose and justify.
- Seeding the centered offset only for the display-first entry (don't regress the existing "open an existing variant" path, which should keep seeding from the display's stored defaults). Consider a parameter/initializer on `VariantDetailView` or a coordinator-held "pending centered offset".
- Demoting manual creation: move "Custom size…" into a menu / secondary button so the primary affordance is "Send to Display".
- Empty state: if no displays exist, guide the user to scan/add (link the discovery flow) rather than dead-ending.

Testing: UI wiring is largely non-unit-testable, but keep any new pure logic (offset seeding/clamping) in testable helpers and cover them (mirror #0063). Mandatory verification is both-platform builds plus the on-device/behavioral checks below. Manual/behavioral: pick a display with a non-matching aspect ratio → preview is aspect-fit and centered (letterboxed via offset, not stretched); send lands centered on the display; manual offset nudges can't push the image off-canvas; the created variant persists and reopens correctly; manual "Custom size…" still works; verified on both iOS and macOS.
