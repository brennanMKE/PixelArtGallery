## Description

The UX is inconsistent and unpolished. There is no shared design system — every screen inline-styles itself, so corner radii (6/8/12), gray fills (0.08/0.1/0.2), `.foregroundColor` vs `.foregroundStyle`, and button affordances all drift. The item detail screen shows a confusing static "Original Image" header (reads like the item name), the photo is letterboxed in a short gray box, and — most importantly — variants have **no visual preview anywhere**: rows are text + an arrow, so the pixel art the app produces is never shown.

## Expected behavior

A consistent, native-feeling UI ("Native + light personality"): a thin design-token layer (one accent, consistent card radius/spacing, monospaced dimension labels), reusable components (section header, status banner, empty state, info row), and real pixel-art thumbnails for variants. The item detail screen shows the photo properly and lists variants as cards with pixel thumbnails. No behavior/schema changes; all existing features and tests remain green.

## Notes

- Full plan (approved): scoped in 5 phases — foundation (Theme/Components/variant-thumbnail helper), item detail + gallery list, variant detail + export sheet, form dedup + grid polish + dead-code, then cross-cutting token adoption + verification.
- Key new capability: render `Variant.pixelGridData` (RGBA8888) → `CGImage` (nearest-neighbor) by extracting the existing logic from `Export/VariantExporter.swift` into a shared `PixelGrid` helper, reused by a new `VariantThumbnailView`.
- Survey found: no `ViewModifier`/`ButtonStyle`/theme file exists; `ExportPickerView` is a structural outlier (no `Form`/`NavigationStack`); `VariantCreationView`/`VariantEditDimensionsView` duplicate their form + `1...512` validation; `PixelGridView` has a stray `.navigationTitle` and an O(w×h) per-cell border draw; `GalleryModels.swift` mocks appear unused.

## Root cause

The app grew feature-by-feature with no shared styling, so every screen inline-styled itself (radii 6/8/12, gray fills 0.08/0.1/0.2, mixed `.foregroundColor`/`.foregroundStyle`, one-off buttons and banners). The item detail screen carried a static "Original Image" header that read like the item name, and there was no code path anywhere to turn a `Variant.pixelGridData` into an image — so variants were never previewed.

## Fix

Introduced a thin design-token layer and reusable components, added variant thumbnails, and redesigned the screens (see the code commit body for the per-screen breakdown). Key structural pieces:
- `Theme` (spacing/radius tokens + `.card()`), `Components` (`SectionHeader`, `StatusBanner`, `EmptyStateView`, `InfoRow`).
- Shared `PixelGrid.makeCGImage(scale:)` extracted from `VariantExporter` (which now reuses it) and rendered by the new `VariantThumbnailView` (nearest-neighbor, off-main).
- `GalleryDetailView` rebuilt as a `List`: photo shown at natural aspect ratio, variants as cards with pixel thumbnails + system chevrons + swipe actions.
- `VariantDetailView`/`ExportPickerView`/`PixelGridView`/`GalleryListView` cleanups (dead state, banners, NavigationStack, batched grid lines, writable alert bindings).
