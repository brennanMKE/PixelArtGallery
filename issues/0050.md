# 0050 — Send button should stay active while sending and let the user tap to stop

| | |
|---|---|
| **Status** | open |
| **Module** | Networking / UI |
| **Platform** | All |
| **First seen** | 2026-07-06 |

## Description

While a send is in flight the Send button is disabled, so the user can't cancel a send that is slow or stuck (e.g. an unreachable display timing out). The button should stay active during sending, switch to a "Stop"/"Cancel" affordance, and tapping it again should cancel the in-flight send.

## Steps to reproduce

1. Open a variant's detail view and pick a display.
2. Tap **Send Now**.
3. While it shows "Sending…", observe the button is disabled — there's no way to stop it. If the display is unreachable, the user must wait for the connection timeout.

## Expected behavior

- The button remains tappable while a send is in progress.
- During a send it reads as a stop action (e.g. "Stop" / "Cancel", with a stop icon).
- Tapping it while sending cancels the in-flight send promptly and returns the UI to the idle "Send Now" state (no success/error flash for a user-initiated cancel, or a neutral "Send cancelled" message).

## Actual behavior

The button is `.disabled(isSending || selectedDisplay == nil)` (`UI/VariantDetailView.swift`), so it is inert for the whole send. The send runs in a detached `Task` in `handleSendToDisplay` whose handle isn't retained, so there is nothing to cancel.

## Notes

Implementation sketch:

- Retain the send `Task` in `@State` (e.g. `sendTask`) so it can be cancelled; toggle behavior in `handleSendToDisplay` — if a send is active, call `sendTask?.cancel()` instead of starting a new one.
- Keep the button enabled while `isSending` (still disable when `selectedDisplay == nil`); swap the label/icon on `isSending`.
- `FTDisplayClient.send` must honor cancellation: check `Task.isCancelled` / use cancellation-aware awaits around the `NWConnection` setup and send, and cancel the connection on cancel. Map a cancelled send to a distinct outcome (not `sendFailed`) so the UI can show a neutral "cancelled" state rather than an error.
- The layer stepper is currently disabled during send (#0047); decide whether it stays disabled while a cancellable send is in flight (likely yes).
